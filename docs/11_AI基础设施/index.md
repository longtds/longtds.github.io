# 11_AI基础设施 · 概述

> 大模型时代，AI 基础设施成为新的数据中心核心。从 GPU 服务器到推理服务，再到成本优化，是 SRE 必修课。

## 一、AI 基础设施发展简史

| 阶段 | 时期 | 标志 |
|:---|:---:|:---|
| 单卡训练 | 2012-2017 | AlexNet/ResNet，单 GPU 训练 |
| 多卡并行 | 2017-2020 | Horovod/AllReduce，多机多卡 |
| 千卡集群 | 2020-2022 | GPT-3 训练（1024 V100） |
| 万卡时代 | 2023-至今 | GPT-4/Llama 训练（25000+ H100） |
| 推理爆发 | 2023-至今 | ChatGPT、vLLM、SGLang |

## 二、AI 基础设施分层

```
┌────────────── 应用层 ──────────────┐
│  对话/RAG/Agent/智能体             │

├────────────── 平台层 ──────────────┤
│  KServe / Seldon / Triton          │
├────────────── 调度层 ──────────────┤

├────────────── 框架层 ──────────────┤
│  PyTorch / TensorFlow / JAX        │
├────────────── 算力层 ──────────────┤
│  CUDA / ROCm / 昇腾 / 寒武纪        │
├────────────── 硬件层 ──────────────┤
│  GPU / NPU / TPU + NVLink / RDMA   │
└────────────────────────────────────┘
```

## 三、训练 vs 推理

| 维度 | 训练 | 推理 |
|:---|:---|:---|
| **算力需求** | 极致（千卡-万卡）| 中等（单卡-数卡） |
| **持续时间** | 数周-数月 | 实时请求 |
| **优化目标** | 训练吞吐 | 延迟 + 吞吐 + 成本 |
| **网络** | RDMA 必须 | TCP 即可 |
| **存储** | 高带宽并行文件 | 模型缓存 |
| **资源利用** | 接近 100% | 通常 30-60% |
| **故障容忍** | 检查点 + 重启 | 多副本即时切换 |

## 五、关键指标

| 指标 | 含义 | 重要性 |
|:---|:---|:---|
| **TTFT** | Time To First Token，首字延迟 | 用户体验关键 |
| **TPOT** | Time Per Output Token，每词延迟 | 流式体验 |
| **TPS** | Tokens Per Second（吞吐） | 容量规划 |
| **GPU Util** | GPU 利用率 | 成本效率 |
| **KV Cache 命中** | 缓存命中率 | 优化效果 |
| **OOM 率** | 显存溢出率 | 稳定性 |

## 六、学习路径

1. **GPU 基础** → CUDA 编程模型、显存层次、NVLink
2. **推理框架** → 至少精通 vLLM
3. **服务架构** → Continuous Batching、PD 分离
4. **K8s 集成** → GPU Operator、KServe
5. **可观测性** → Prometheus + GPU 指标
6. **成本优化** → 量化、推测解码、缓存

> 📖 AI Infra 是当下最热的运维方向，技术更新快，但**底层原理稳定**——GPU + 网络 + 调度。
