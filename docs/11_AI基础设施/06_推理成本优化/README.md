# 推理成本优化

## GPU 利用率提升

| 策略 | 效果 | 实现 |
|:---|:---:|:---|
| Continuous Batching | 2-3x 吞吐 | vLLM/TGI 默认支持 |
| 动态批处理 | 1.5x 吞吐 | 队列合并请求 |
| 量化（INT4） | 4x 显存节省 | AWQ/GPTQ |
| 解耦推理 | 2x GPU 利用 | prefill/decode 分离 |

## 冷热模型分层

```
热模型（高频）：常驻 GPU，低延迟 → L40S/A100
温模型（中频）：GPU 共享，可冷启动 → 量化部署
冷模型（低频）：CPU 推理/按需加载 → 存于 S3
```

## 弹性策略

- HPA 基于 GPU 利用率和请求排队长度
- 预测性扩缩（根据历史流量模式）
- 模型缓存预热（避免冷启动）

## 成本公式

```
推理总成本 = GPU 实例成本 + 存储成本 + 网络成本
每 Token 成本 = GPU 成本 / 总 Token 数
优化目标：降低每 Token 成本，而非单个 GPU 成本
```
