## 二、PD 分离架构（Prefill-Decode）

```
传统:  Prefill (计算密集) + Decode (访存密集) 共一卡
              ↓ 资源利用率低、互相干扰

PD 分离:
        ┌─── Prefill Pool (计算优化卡)
请求 ─→ │
        └─── Decode Pool (大显存卡)
              ↓ KV Cache 跨节点传输（RDMA）

优势: GPU 利用率 +40%, 长序列友好
代表: vLLM disaggregated、llm-d、Mooncake
```

## 三、推理成本优化

### 3.1 量化（Quantization）

| 精度 | 显存 | 性能 | 损失 |
|:---|:---:|:---:|:---|
| FP16 | 1x | 1x | 基线 |
| INT8 | 0.5x | 1.5x | < 1% |
| FP8 | 0.5x | 2x（H100） | < 1% |
| INT4 (AWQ/GPTQ) | 0.25x | 2-3x | 1-3% |

**推荐**：H100 用 FP8，A100 用 INT8/INT4，A10 用 AWQ。

### 3.2 模型选择

```
通用场景:    Llama-3.x 系列 (8B/70B/405B)
中文场景:    Qwen3 / DeepSeek 系列
代码场景:    DeepSeek-Coder / Qwen-Coder
长文本:      具备 128K+ context 的模型
端侧场景:    Llama-3.x-3B / Qwen3-1.8B
推理场景:    DeepSeek-R1 / QwQ
```

### 3.3 多级缓存

```
请求频率高，多用户共享前缀
   ↓
Prefix Caching (LRU 缓存常用前缀的 KV)
   ↓ 命中率 30%+
GPU 显存 KV → CPU DRAM → SSD（三级存储）
   ↓
成本下降 50%+
```
