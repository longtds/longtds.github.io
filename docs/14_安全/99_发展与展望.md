# 安全发展与展望

> AI 重塑了攻击和防御，零信任和供应链安全成为 2025+ 的两大主轴。

## 一、零信任全面铺开

```
传统:   "内网可信"           → 内部攻击横行
零信任:  "Never Trust, Always Verify"
         每次请求都验证身份 + 设备 + 上下文
```

| 组件 | 代表方案 |
|:---|:---|
| ZTNA（零信任接入） | Cloudflare Access、Pomerium、Tailscale |
| SASE（安全访问服务边缘） | Zscaler、Netskope、Cato Networks |
| 微分段 | Cilium NetworkPolicy、Illumio |
| 设备合规检查 | Microsoft Intune、Jamf |

**国内**：奇安信、深信服、腾讯零信任、阿里云 IDaaS。

## 二、供应链安全（最大新威胁）

### 2.1 重大事件

| 年份 | 事件 | 影响 |
|:---|:---|:---|
| 2020 | SolarWinds | 18000 客户被植入后门 |
| 2021 | Log4Shell (CVE-2021-44228) | 全球紧急修复 |
| 2024 | XZ Utils 后门 (CVE-2024-3094) | 上游开源仓库被渗透 |
| 2024 | npm/PyPI 仿冒包 | 大规模发布恶意包 |

### 2.2 防护体系

```
源头:   依赖最小化 + 来源验证
        ↓
扫描:   SCA 软件成分分析（Trivy/Snyk/Grype）
        ↓
SBOM:   软件物料清单（CycloneDX/SPDX）
        ↓
签名:   Sigstore (cosign + Fulcio + Rekor)
        ↓
策略:   K8s admission（policy-controller）
        ↓
审计:   完整溯源链路
```

### 2.3 SLSA 框架

```
SLSA L1: 自动化构建
SLSA L2: 版本控制 + 认证构建服务
SLSA L3: 完整构建链路（GitHub Actions / GitLab CI 已支持）
SLSA L4: 双人审查 + 隔离构建
```

## 三、AI 安全（全新战场）

### 3.1 攻击面

| 类型 | 描述 |
|:---|:---|
| **提示词注入** | 让 LLM 突破指令限制 |
| **模型越狱** | 绕过安全护栏 |
| **数据投毒** | 训练数据被污染 |
| **模型窃取** | 提取模型权重 |
| **隐私泄露** | 训练数据被反推 |
| **Agent 滥用** | LLM Agent 被诱导执行恶意操作 |

### 3.2 防御工具

| 工具 | 用途 |
|:---|:---|
| **NVIDIA NeMo Guardrails** | 提示词过滤、内容护栏 |
| **Llama Guard** | Meta 出品的内容审核模型 |
| **LangChain Guardrails** | 多 LLM 通用护栏 |
| **OWASP LLM Top 10** | 风险清单 |

### 3.3 AI 安全标准

- **MITRE ATLAS** — AI 攻击知识库
- **OWASP LLM Top 10** — LLM 应用 OWASP
- **NIST AI RMF** — AI 风险管理框架
- **GB/T 45654** — 国内大模型安全标准

## 五、运行时安全演进

### 5.1 eBPF 安全

```
传统 HIDS:  应用级钩子，可被绕过
eBPF 安全:  内核级钩子，无法绕过 + 性能极低
```

| 工具 | 用途 |
|:---|:---|
| Falco | 运行时异常检测 |
| Tetragon | 实时阻断 + 可观测 |
| Tracee | Aqua 出品 |
| Pixie | 应用级观测 + 安全 |

### 5.2 机密计算

## 十、学习路径

1. **基础**：网络安全 + 加密原理
2. **主机**：Linux 加固 + auditd
3. **云原生**：容器/K8s 安全
4. **供应链**：Trivy + SBOM + Sigstore
5. **运行时**：Falco / Tetragon
6. **体系化**：等保 / ISO 27001 / SOC 体系

> 💡 安全是"反脆弱"工程——攻击者只需要一次成功，防守者必须一直成功。
