# 05_私有云 · 概述

> 私有云让企业获得云的弹性、自助和 API，同时把数据握在自己手里。

## 一、私有云发展简史

| 阶段 | 时期 | 代表 |
|:---|:---:|:---|
| 虚拟化整合 | 2008-2012 | VMware vSphere 主导 |
| 开源云崛起 | 2012-2018 | OpenStack 黄金期 |
| 云原生融合 | 2018-至今 | OpenStack + K8s 双栈 |
| 软硬一体 | 2020-至今 | 一体机、智算云 |

## 二、私有云三层架构

```
┌──────────────────────────────────────┐
│  应用层  │ 业务系统 / 中间件 / 数据库 │
├──────────────────────────────────────┤
│  服务层  │ IaaS API / 控制台 / 调度   │
│         │ OpenStack / K8s / Ceph    │
├──────────────────────────────────────┤
│  基础层  │ 服务器 / 网络 / 存储       │
└──────────────────────────────────────┘
```

## 三、典型方案对比

| 方案 | 厂商/社区 | 特点 | 适用规模 |
|:---|:---|:---|:---|
| OpenStack | 社区/Red Hat | 功能全、复杂、组件多 | 大型企业 |
| KubeVirt + K8s | 社区/Red Hat | K8s 原生管 VM | 云原生为主 |
| vSphere | VMware | 商业、稳定、贵 | 传统企业 |
| ZStack | 国产 | 轻量、易部署 | 中小企业 |
| OpenShift | Red Hat | 商业 K8s + 虚拟化 | 大企业 |

## 五、学习路径

1. **理解 IaaS 模型** → 计算/网络/存储三大资源
2. **掌握 OpenStack 一项** → 推荐从 Nova/Neutron 入手
3. **学会编排** → Heat / Terraform 自动化
4. **跟进趋势** → KubeVirt 替代方案

> 📖 私有云比公有云难做，因为没有"无限资源"做缓冲。
