# 私有云发展与展望

> OpenStack 进入"成熟稳定"阶段，新增长点来自 K8s + KubeVirt 的融合方案。

## 一、OpenStack 现状

| 维度 | 状态 |
|:---|:---|
| 大版本节奏 | 每年一个版本（Antelope/Bobcat/Caracal） |
| 部署热度 | 大企业稳定使用，新增减少 |
| 主流发行版 | Red Hat OSP / 华为 FusionSphere / Mirantis |
| 容器化部署 | Kolla-Ansible / OpenStack-Helm 主流 |

**OpenStack 适合场景**：
- 大规模 VM 工作负载（万核以上）
- 多租户隔离要求高
- 已有运维团队，沉没成本大

**OpenStack 不适合场景**：
- 容器为主的新建项目
- 中小规模（< 100 节点）
- 团队不超过 5 人

## 二、KubeVirt 崛起

KubeVirt 把 VM 当 K8s 资源管理：

```yaml
apiVersion: kubevirt.io/v1
kind: VirtualMachine
metadata:
  name: my-vm
spec:
  template:
    spec:
      domain:
        devices:
          disks: [...]
        resources:
          requests:
            memory: 4Gi
            cpu: 2
```

| 对比维度 | OpenStack | KubeVirt |
|:---|:---|:---|
| 控制平面 | 自研多组件 | 复用 K8s |
| 学习曲线 | 陡峭 | 平缓（K8s 知识可复用） |
| 容器 + VM 混合 | 难 | 天然支持 |
| 生态 | 成熟但封闭 | K8s 生态全部可用 |

**适用场景**：新建私有云、AI 训练平台、边缘节点。

## 三、超融合（HCI）

```
传统:
├── 计算节点（虚拟化）
├── 网络设备（核心/接入）
└── 存储设备（SAN/NAS）

超融合:
└── 通用 x86 服务器 × N
    ├── 计算（虚拟化）
    ├── 网络（虚拟交换）
    └── 存储（分布式块存储）
```

| 厂商 | 产品 | 技术栈 |
|:---|:---|:---|
| Nutanix | AHV | KVM + 自研分布式存储 |
| VMware | vSAN | ESXi + vSAN |
| SmartX | SMTX OS | KVM + ZBS |
| 华为 | FusionCube | KVM + FusionStorage |
| 深信服 | aSV | KVM + EDS |

## 四、智算云（AI 私有云）

为 AI 训练/推理优化的私有云：

| 特性 | 通用云 | 智算云 |
|:---|:---|:---|
| 网络 | 25G/100G | 400G + RDMA |
| GPU 互联 | PCIe | NVLink + IB Switch |
| 存储 | 块/文件 | 高带宽并行文件系统 |
| 调度 | VM 粒度 | Gang Scheduling |
| 监控 | CPU/Mem | GPU 利用率、TFLOPS |

**国内代表**：商汤 AIDC、字节火山、阿里 PAI、华为 ModelArts。

## 五、Bare Metal as a Service (BMaaS)

物理机即服务，API 管理裸金属：

| 方案 | 用途 |
|:---|:---|
| Ironic (OpenStack) | 老牌、功能全 |
| Tinkerbell (Equinix) | 现代、API 友好 |
| Metal³ | K8s 原生，CAPI 集成 |
| ZTP (Zero Touch Provisioning) | 网络设备零配置部署 |

AI 训练对裸金属诉求很强（避开虚拟化损耗 + 直接访问 RDMA/GPU）。

## 六、未来 3-5 年展望

| 方向 | 预测 |
|:---|:---|
| **K8s 统一控制面** | OpenStack 多组件被 K8s API 取代 |
| **KubeVirt 普及** | 中小规模私有云首选 |
| **裸金属 API 化** | 物理机部署像虚拟机一样快 |
| **智算云爆发** | 每个大企业都建 AI 私有云 |
| **混合多云** | 私有 + 公有混合调度，OSS 工具成熟 |
| **国产化** | 信创私有云成为政府/金融标配 |

## 七、对运维的影响

| 变化 | 运维新要求 |
|:---|:---|
| KubeVirt 化 | K8s 运维即私有云运维 |
| 智算云 | 必备 GPU + RDMA + 调度知识 |
| BMaaS | iPXE/MAAS/Tinkerbell |
| 多云管理 | Crossplane/Terraform 跨云能力 |
| 信创栈 | 国产芯片/OS/虚拟化全链路 |

## 八、学习路径

1. **基础**：OpenStack 核心组件原理
2. **进阶**：KubeVirt + Ceph + Calico
3. **方向**：智算云（推荐）或 BMaaS
4. **国产化**：openEuler + 鲲鹏/海光 + 国产 VMM

> 💡 私有云的终极形态是"K8s API + 一切皆 CRD"，控制平面收敛。
