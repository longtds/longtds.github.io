# DevOps 发展与展望

> DevOps 正在向 **平台工程（Platform Engineering）** 演进，自助式 + 黄金路径成为新标准。

## 一、平台工程崛起（最大趋势）

### 1.1 为什么需要平台工程

```
DevOps 早期理想:    "You build it, you run it"
                     ↓
现实问题:           开发者要懂 K8s/IaC/CI/CD/监控...负担过重
                     ↓
平台工程方案:        基础设施团队提供"内部开发者平台 (IDP)"
                     ↓
最终形态:           开发者通过自助界面/CLI 完成所有运维操作
```

### 1.2 IDP 核心组件

| 模块 | 工具 |
|:---|:---|
| **服务目录** | Backstage（事实标准） |
| **黄金路径** | 模板化项目脚手架 |
| **自助门户** | Web UI + CLI |
| **基础设施抽象** | Crossplane / Humanitec |
| **集成 CI/CD** | GitHub Actions / Tekton |
| **可观测性** | 一键接入监控/日志 |

### 1.3 主流方案

| 方案 | 类型 |
|:---|:---|
| Backstage | 开源（Spotify）|
| Port | 商业、轻量 |
| Humanitec | 商业、抽象层强 |
| Cortex | 商业、服务目录 |
| Mia-Platform | 欧洲流行 |

## 二、CI/CD 演进

### 2.1 Pipeline 现代化

```
Jenkinsfile (脚本化)
   ↓
GitHub Actions / GitLab CI (声明式)
   ↓
Tekton / Argo Workflows (K8s 原生)
   ↓
Dagger (代码即 Pipeline)
```

### 2.2 Dagger（新范式）

```python
# Pipeline 用编程语言写
import dagger

async def build(client):
    src = client.host().directory(".")
    image = client.container().from_("python:3.12").with_directory("/app", src)
    await image.with_exec(["pytest"]).sync()
```

优势：本地可调试、跨 CI 工具复用、强类型。

### 2.3 GitHub Actions / GitLab CI 主导

| 维度 | Jenkins | GitHub Actions | GitLab CI |
|:---|:---|:---|:---|
| 学习成本 | 高 | 低 | 低 |
| 主机 | 自建为主 | SaaS + Runner | SaaS + Runner |
| 生态 | 插件多但老 | Marketplace 活跃 | 内嵌完整 |
| 适合 | 大型企业 | 开源 / 中小 | 一体化平台 |

## 三、GitOps 主流化

### 3.1 GitOps 核心原则

```
1. 声明式: 系统期望状态用 YAML 描述
2. 版本化: 所有变更通过 Git
3. 自动同步: 控制器拉取并应用
4. 自动协调: 漂移自动修复
```

### 3.2 主流工具

| 工具 | 厂商 | 特点 |
|:---|:---|:---|
| **Argo CD** | Intuit/社区 | 单集群强、UI 好 |
| **Flux** | Weaveworks | 多租户、推送模型 |
| **Rancher Fleet** | SUSE | 多集群分发 |
| **Jenkins X** | Jenkins | Jenkins + GitOps |

### 3.3 Progressive Delivery

```
Argo Rollouts / Flagger
   ↓
按流量百分比灰度
   ↓
Prometheus 指标自动判断 → 失败回滚 / 成功推进
```

## 四、IaC 演进

### 4.1 Terraform → OpenTofu 之争

```
2014: Terraform 开源
2023: HashiCorp 改 BSL 许可证 → 引发 Fork
2023.8: OpenTofu 成立（Linux Foundation）
2024+: OpenTofu 加速创新，企业逐步迁移
```

### 4.2 新一代 IaC

| 工具 | 语言 | 特点 |
|:---|:---|:---|
| Terraform / OpenTofu | HCL | 主流，多云覆盖 |
| Pulumi | Python/Go/TS | 用编程语言 |
| Crossplane | YAML（K8s CRD） | K8s 原生 IaC |
| Winglang | 专用 DSL | 跨云抽象 |
| CDKTF | TypeScript | TF + 代码 |

### 4.3 Crossplane（K8s 原生）

```yaml
# 在 K8s 里声明云资源
apiVersion: rds.aws.upbound.io/v1beta1
kind: Instance
metadata:
  name: production-db
spec:
  forProvider:
    instanceClass: db.t3.medium
    engine: postgres
```

## 六、安全左移（DevSecOps）

```
传统:  开发 → 测试 → 安全审计 → 上线
        ↓ 安全是瓶颈
DevSecOps:  安全融入每个环节
        - 代码提交：SAST 扫描
        - 构建：SCA 依赖扫描
        - 镜像：Trivy 漏洞扫描
        - 部署：策略引擎（OPA）
        - 运行：Falco 运行时安全
```

## 八、对运维的影响

| 变化 | 运维新要求 |
|:---|:---|
| 平台工程 | 学 Backstage、设计黄金路径 |
| GitOps | Argo CD / Flux 运维专精 |
| OpenTofu 迁移 | 状态文件迁移、模块适配 |
| AI 工具 | Copilot/Claude Code 提效 |

## 九、学习路径

1. **基础**：Git + Shell + Linux
2. **CI/CD**：选一个深度（GitLab CI 或 GitHub Actions）
3. **IaC**：OpenTofu/Terraform + Ansible
4. **K8s + GitOps**：Argo CD 实战
5. **平台工程**：Backstage 部署、模板设计
6. **AI 提效**：Copilot/Claude Code/Cursor

> 💡 DevOps 的下一站不是更好的工具，而是**让开发者根本不需要懂 DevOps**——这就是平台工程的目标。
