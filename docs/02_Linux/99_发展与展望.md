# Linux 发展与展望

> Linux 正从"通用服务器 OS"走向"云原生 OS + AI 基础设施 OS"两条路径。

## 一、当前技术趋势

### 1.1 eBPF 革命

eBPF 让内核可编程，无需修改源码即可注入观测/网络/安全逻辑：

| 工具 | 用途 |
|:---|:---|
| Cilium | 替代 iptables 的 K8s 网络与安全 |
| Pixie | 零侵入应用观测 |
| Tetragon | 运行时安全 |
| Parca | 持续性能分析 |
| bpftrace | 实时内核追踪 |

### 1.2 不可变 Linux

容器化推动 OS 自身也走向不可变：

| 发行版 | 厂商 | 定位 |
|:---|:---|:---|
| Talos Linux | Sidero | K8s 专用，纯 API 管理 |
| Bottlerocket | AWS | 容器宿主机 |
| Flatcar | Microsoft | CoreOS 后继 |
| Fedora Silverblue | Red Hat | 桌面不可变 |

### 1.3 cgroup v2 全面铺开

| 改进 | 价值 |
|:---|:---|
| 统一层级 | 不再需要 v1 多个挂载点 |
| 内存 QoS | memory.low/high/max 三档 |
| IO 控制 | 真正可工作的 blkio |
| 进程数限制 | pid.max 防 fork 炸弹 |

K8s 1.25+ 默认使用 cgroup v2。

### 1.4 io_uring 异步 IO

io_uring 是 Linux 5.1+ 引入的高性能异步 IO 框架，比 epoll + AIO 性能高 50-100%。
PostgreSQL 17、ScyllaDB、Ceph 都已集成。

## 二、内核新特性时间线

| 内核版本 | 年份 | 重大特性 |
|:---|:---:|:---|
| 5.4 LTS | 2019 | WireGuard 内置 |
| 5.10 LTS | 2020 | io_uring 稳定 |
| 5.15 LTS | 2021 | NTFS3、in-kernel SMB |
| 6.1 LTS | 2022 | Rust for Linux 进入主线 |
| 6.6 LTS | 2023 | EEVDF 调度器替代 CFS |
| 6.12 LTS | 2024 | sched_ext 可插拔调度 |

## 三、Rust in Linux

Linux 6.1 起允许用 Rust 编写驱动。当前进展：

| 模块 | 状态 |
|:---|:---|
| NVMe 驱动 | 实验性 |
| Apple GPU 驱动 (Asahi) | 已合入 |
| binder（Android）| 重写中 |
| 文件系统层 | API 准备中 |

**意义**：内存安全 + 并发安全，减少内核 60% 的安全漏洞类别。

## 四、AI 基础设施 OS

GPU 服务器对 OS 有专门要求：

| 需求 | 应对 |
|:---|:---|
| NVIDIA 驱动开箱即用 | Ubuntu/RHEL + DGX OS |
| Topology-Aware 调度 | NUMA + GPU 拓扑感知 |
| InfiniBand/RoCE 加速 | RDMA over Ethernet |
| 大页内存 | Transparent Huge Pages |
| 实时性 | PREEMPT_RT 内核 |

## 六、未来 3-5 年展望

| 方向 | 预测 |
|:---|:---|
| **eBPF 渗透** | 80% 的可观测/安全/网络工具基于 eBPF |
| **Rust 占比** | 内核 5-10% 代码用 Rust 重写 |
| **不可变 OS** | 云原生场景全面普及 |
| **AI 加速** | 内核原生支持 GPU 调度、NPU 异构 |
| **机密计算** | TDX/SEV-SNP/CCA 成为标配 |
| **国产化** | openEuler 进入服务器市场前三 |

## 七、对运维的影响

| 变化 | 运维新要求 |
|:---|:---|
| eBPF 工具链 | 学会 bpftrace/bcc/Cilium |
| cgroup v2 | 重新审视 QoS 策略 |
| io_uring | 应用调优新维度 |
| Rust 模块 | 看懂内核 Rust 代码 |
| 不可变 OS | 抛弃"登机改文件"思维 |
| 国产 OS | 信创栈的全链路排查能力 |

## 八、推荐学习路径

1. **打牢基础**：systemd、cgroup v2、网络栈
2. **掌握 eBPF**：从 bpftrace 入门，进阶到 libbpf-bootstrap
3. **跟进 LTS**：每两年升级一次内核认知
4. **关注国产**：openEuler/OpenAnolis 已是必备技能

> 💡 Linux 永远不会过时，但**用 Linux 的方式**每 5 年就会换一次。
