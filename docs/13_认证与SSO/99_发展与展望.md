# 发展与展望

> 身份认证正在被重塑：**Passkey/WebAuthn 全民化 + 无密码 + 零信任主流 + 工作负载身份(SPIFFE)成标配 + Verifiable Credentials/DID + AI Agent 身份 + 国密+等保强约束 + 主权身份(Sovereign Identity) + AI 反钓鱼**。

## 一、12 大趋势

```
1. Passkey / WebAuthn 取代密码 (2026-2028 主流)
2. 零信任 / SASE 取代 VPN (2026-2028)
3. SPIFFE / mTLS 成 K8s 默认 (2027+)
4. Verifiable Credentials / DID 兴起 (W3C 标准)
5. Agentic AI 身份新挑战 (Agent 代用户操作)
6. MCP 工具调用身份委托
7. UEBA + Risk-based MFA 普及
8. ReBAC (OpenFGA) 复杂权限主流
9. 国密 + 等保三级 强约束 (中国)
10. Sovereign Identity 国家战略 (主权身份)
11. AI 反钓鱼 (Deepfake / 语音克隆)
12. Confidential Computing + 身份 (TEE 证明)
```

## 五、Verifiable Credentials / DID

```
W3C 标准:
  - DID (Decentralized Identifier)
  - VC (Verifiable Credential)
  - 自主主权身份 (SSI)

场景:
  - 学历认证 (毕业证 VC)
  - 工作经历 (公司签发 VC)
  - 身份证 (政府签发)
  - 医疗 (HIPAA 兼容 VC)
  - 数字钱包 (Wallet)

工具:
  - Microsoft Entra Verified ID
  - Hyperledger Indy / Aries
  - Trinsic (商业)
  - Veramo (开源)

中国:
  - 长安链 (DID)
  - 蚂蚁链 (区块链身份)
  - 国家 eID (公安部) → 数字身份证
  
5 年判断:
✅ VC / DID 政务先行 (2027+)
✅ 数字身份证全国推广
✅ Web3 身份 (有限场景)
```

## 六、Agentic AI 身份挑战

```
新场景:
  AI Agent 代用户操作
    - Coding Agent (Cursor / Claude Code)
    - Customer Agent
    - RPA Agent
    - AIOps Agent

问题:
  - Agent 身份 ≠ 用户身份
  - Agent 操作 审计 (代谁 / 干啥)
  - Agent 委托 (Token Exchange)
  - Agent 权限 (最小化, 不可越权)
  
解法:
  - Token Exchange (RFC 8693) ⭐
  - act claim (谁代谁)
  - Audit 双层 (Agent + User)
  - 显式同意 / 步骤审批
  
工具:
  Anthropic MCP + auth (探索)
  Auth0 Agent (商业试点)
  
5 年判断:
✅ Token Exchange 普及 (2026-2027)
✅ Agent 身份成新标准 (2027-2028)
✅ act claim + Audit 全链 (2027+)
```

## 八、ReBAC (OpenFGA) 主流

```
RBAC:    角色驱动 (简单)
ABAC:    属性驱动 (OPA, 灵活但复杂)
ReBAC:   关系驱动 (Google Zanzibar, 业务自然)

场景:
  - "Alice 可读 Doc1, 因为是 Doc1 owner"
  - "Bob 可写, 因为加入了 Doc1 的协作者"
  - 文档 / 资源 / 项目 / 团队 复杂关系

工具:
  Google Zanzibar (内部)
  OpenFGA ⭐ (Auth0 开源)
  SpiceDB (商业 + 开源)
  Cerbos
  
5 年判断:
✅ OpenFGA 进入主流 (2026-2027)
✅ 大型 B2B SaaS 必修
```

## 十五、结语

```
身份认证正在被 4 大力量重塑:
  1. Passkey - 无密码取代 30 年密码体系
  2. 零信任 - 网络边界消解
  3. AI - Agent 身份 + 反钓鱼 + UEBA
  4. 主权 - 国家数字身份 + 国密 + 等保

从 LDAP → SSO → 联邦 → OIDC → Passkey → DID
从 静态 SSH key → CA → 短期证书 → SPIFFE → mTLS
从 静态密码 → MFA → WebAuthn → Liveness
从 RBAC → ABAC → ReBAC → AI 策略
从 网络边界 → 零信任 → SASE → Sovereign
从 人 → 服务 → Agent 身份
从 国际 → 国产 → 国密 → 主权

工程师该做的:
  1. 别守 LDAP / 静态密码 - 必修 Passkey + Vault Dynamic
  2. K8s 必走 OIDC + Workload Identity
  3. SPIFFE + Service Mesh 是下一波分水岭
  4. 国密 + 等保 + 阿里 IDaaS 是央企必修
  5. Teleport + Vault + WebAuthn 是当下最值学组合
  6. Agent 身份 + Token Exchange 是 AI 时代核心
  7. OpenFGA / ReBAC 业务复杂场景必修
  8. 钓鱼演练 + 反 Deepfake 培训
  9. 平台工程 + Backstage 自服务 是 IAM 趋势
  10. 不学零信任 + 国密的 SSO 工程师 5 年内淘汰
```

> 📖 **核心判断**：身份认证的未来 = **Passkey 取代密码 + 零信任 / SASE 取代 VPN + SPIFFE/mTLS K8s 默认 + Vault 全栈动态 + Token Exchange + Agent 身份 + ReBAC + UEBA + Anti-Deepfake + 国密 + 等保 + Sovereign Identity**。把这 12 条线钉死, 5 年内不愁。**身份不会变, 但实现方式正经历 30 年最大变革** —— 这是这一代身份工程师最大的机遇与分水岭。
