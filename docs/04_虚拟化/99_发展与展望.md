# 虚拟化发展与展望

> 虚拟化正在向两个方向分化：极致轻量（Firecracker/MicroVM）和极致安全（机密计算）。

## 一、轻量级虚拟化崛起

| 技术 | 厂商 | 启动时间 | 内存开销 | 用途 |
|:---|:---|:---:|:---:|:---|
| Firecracker | AWS | < 125ms | 5MB | Lambda、Fargate |
| Cloud Hypervisor | Intel/Microsoft | < 200ms | 10MB | 通用云 |
| Kata Containers | 社区 | ~1s | 30MB | K8s 安全沙箱 |
| gVisor | Google | < 500ms | 用户态 | Cloud Run |

**意义**：传统 VM 启动慢、密度低，无法支持 Serverless。MicroVM 把 VM 启动降到容器级别。

## 二、GPU 虚拟化

### 2.1 三种主流方案

| 方案 | 厂商 | 切分粒度 | 性能 |
|:---|:---|:---|:---|
| **vGPU** (软件) | NVIDIA Grid | 时间片切分 | 90% |
| **MIG** (硬件) | NVIDIA A100/H100 | 硬件分区（最多7个）| 100% |
| **SR-IOV** | AMD/Intel | PCI 物理功能 | 100% |
| **PCI 直通** | 通用 | 整卡独占 | 100% |

### 2.2 K8s 中的 GPU 虚拟化

```yaml
# MIG 方式
resources:
  limits:
    nvidia.com/mig-1g.5gb: 1   # 1/7 GPU
    
# 时间片方式
resources:
  limits:
    nvidia.com/gpu: 1   # GPU Operator 配置 timeslicing
```

## 三、机密计算（Confidential Computing）

```
传统 VM:  Host 可以读 VM 内存 → 多租户不安全
                ↓
机密 VM:  内存硬件加密 → Host 也读不到
```

| 技术 | 厂商 | 内存加密 | 完整性证明 |
|:---|:---|:---|:---|
| Intel TDX | Intel | ✅ | TD Quote |
| AMD SEV-SNP | AMD | ✅ | Attestation Report |
| ARM CCA | ARM | ✅ | Realm Attestation |
| NVIDIA Hopper Confidential | NVIDIA | ✅ GPU 内存加密 | NVTrust |

**应用场景**：金融多方计算、AI 模型保护、医疗数据训练。

## 四、虚拟化 + 容器融合

```
传统:  VM    OR    容器  （二选一）
            ↓
现代:  VM 里跑容器 + 容器用 MicroVM 隔离
            ↓
未来:  统一资源池（CRI + KubeVirt）
```

| 项目 | 价值 |
|:---|:---|
| KubeVirt | K8s 管理 VM（CRD + virt-launcher） |
| Kata Containers | 看起来是容器，实际是 MicroVM |
| Confidential Containers | Kata + TDX/SEV |

## 五、Wasm 作为新虚拟化层

WebAssembly 正在成为容器之后的下一层：

| 维度 | 容器 | Wasm |
|:---|:---|:---|
| 启动时间 | 100ms | < 1ms |
| 镜像大小 | MB-GB | KB-MB |
| 安全 | 共享内核 | 沙箱隔离 |
| 跨架构 | 多镜像 | 一份代码 |

**代表项目**：
- **WasmEdge** — CNCF 沙箱
- **Spin** — Fermyon 框架
- **Krustlet** — K8s 直接跑 Wasm

## 七、未来 3-5 年展望

| 方向 | 预测 |
|:---|:---|
| **MicroVM 普及** | Serverless 场景 90%+ 使用 |
| **机密计算** | 金融/医疗强制要求 |
| **GPU MIG 普及** | 推理场景标配 |
| **Wasm 渗透** | 边缘/插件场景规模化 |
| **KubeVirt 替代 OpenStack** | 一些场景 K8s 同时管 VM + 容器 |
| **零损耗虚拟化** | DPU 卸载到完全等同物理机 |

## 八、对运维的影响

| 变化 | 运维新要求 |
|:---|:---|
| MicroVM | 调试工具链变化（firecracker-ctr 等） |
| GPU MIG | nvidia-smi mig + K8s Device Plugin |
| 机密计算 | 远程证明（Attestation）流程 |
| KubeVirt | 同时具备 K8s 和 VM 运维技能 |
| Wasm 沙箱 | 容器外的新部署方式 |

## 九、学习路径

1. **基础**：精通 KVM/QEMU/libvirt
2. **网络**：Open vSwitch、SR-IOV
3. **GPU**：MIG/vGPU/Device Plugin
4. **前沿**：Kata/Firecracker/KubeVirt

> 💡 容器没有取代虚拟机，反而让虚拟机进化得更快了。
