# 08_认证与SSO · 概述

> 在多系统时代，统一身份认证（SSO）是企业 IT 的中枢神经系统。一处登录，处处通行。

## 一、企业认证发展简史

| 阶段 | 时期 | 方案 |
|:---|:---:|:---|
| 单系统密码 | 1990s | 各应用独立账号 |
| LDAP 域控 | 2000s | AD / OpenLDAP |
| SAML SSO | 2005-2015 | 企业 Web SSO 标准 |
| OAuth/OIDC | 2015-至今 | 互联网+移动+API 友好 |
| 零信任 | 2020-至今 | 持续验证、上下文感知 |

## 二、核心协议对比

| 协议 | 用途 | 数据格式 | 主要场景 |
|:---|:---|:---|:---|
| **LDAP** | 目录查询 | LDIF | 企业用户库 |
| **Kerberos** | 网络认证 | Ticket | 域内 SSO |
| **SAML 2.0** | 跨域 Web SSO | XML | 传统企业应用 |
| **OAuth 2.0** | 授权 | JSON | API 访问授权 |
| **OIDC** | 身份层（OAuth 之上）| JWT | 现代 Web/移动 SSO |
| **WebAuthn** | 无密码 | CTAP/FIDO2 | 强认证、生物识别 |

## 三、典型 SSO 流程（OIDC）

```
用户                  应用              IdP (Keycloak)
 │  1.访问应用         │                    │
 ├──────────────────→  │                    │
 │                    │  2.重定向到 IdP     │
 │  3.登录页面          │                    │
 ←──────────────────────────────────────────│
 │  4.输入凭证          │                    │
 ├──────────────────────────────────────→  │
 │  5.回调 + 授权码     │                    │
 ←──────────────────────────────────────────│
 │                    │ 6.授权码换 token    │
 │                    ←────────────────→   │
 │  7.返回应用页面       │                    │
 ←──────────────────────                    │
```

## 四、主流 IdP 方案

| 产品 | 类型 | 协议支持 | 适用 |
|:---|:---|:---|:---|
| **Keycloak** | 开源 | OIDC/SAML/LDAP | 自建首选 |
| **Authelia** | 开源 | OIDC/2FA | 轻量级 |
| **Authentik** | 开源 | 全 | 现代 UI |
| **Okta** | SaaS | 全 | 大型企业 |
| **Auth0** | SaaS | 全 | 互联网应用 |
| **Azure AD / Entra ID** | SaaS | 全 + 域控 | Microsoft 体系 |

## 六、企业最佳实践

1. **统一身份源** → 全公司唯一 IdP，禁止应用自建账户
2. **生命周期自动化** → HR 入离职联动 IdP
3. **强制 MFA** → 至少 SMS + TOTP，敏感系统 WebAuthn
4. **细粒度授权** → 角色 + 属性双重控制（RBAC + ABAC）
5. **审计可追溯** → 所有认证事件 6 个月以上
6. **零信任接入** → 设备状态 + 位置 + 行为评估

## 七、学习路径

1. **协议基础** → 先 OAuth 2.0 + OIDC，再 SAML
2. **部署 Keycloak** → 接入第一个应用
3. **集成实战** → K8s OIDC、GitLab、Grafana、Confluence
4. **进阶** → 多 Realm、Identity Brokering、自定义 SPI

> 📖 SSO 不是技术问题，是**身份治理问题**——技术好做，治理难做。
