# 认证与 SSO 发展展望

> 身份是新的边界，零信任和无密码是确定方向。

## 一、零信任架构（Zero Trust）

```
传统:           内网信任 + VPN
                ↓ 内部攻击横行
零信任:         "永不信任，持续验证"
                每次请求都鉴权
                设备 + 用户 + 上下文综合评估
```

| 组件 | 作用 |
|:---|:---|
| **Identity Provider** | 用户身份认证 |
| **Policy Engine** | 策略评估（允许/拒绝/挑战）|
| **Policy Enforcement Point** | 网关执行决策 |
| **Device Health** | 终端合规检查 |
| **Risk Engine** | 行为/位置/时间异常检测 |

## 二、无密码认证（Passwordless）

| 技术 | 标准 | 用户体验 |
|:---|:---|:---|
| **Passkey** | WebAuthn + FIDO2 | 生物识别，本地存储 |
| **Magic Link** | 邮件链接 | 一次性登录 |
| **Push Notification** | 推送批准 | 手机点确认 |
| **QR Code** | 扫码授权 | 跨设备登录 |

**2025 后**：Apple/Google/Microsoft 都已支持 Passkey 同步，消费场景普及。

## 三、机器身份（Machine Identity）

人类身份只是一半，服务/容器/Agent 也需要身份：

| 标准 | 用途 |
|:---|:---|
| **SPIFFE** | 统一服务身份框架 |
| **SPIRE** | SPIFFE 实现 |
| **mTLS** | 服务间互信 |
| **OIDC Workload Identity** | K8s ServiceAccount 联邦 |

K8s 1.30+ 支持 Workload Identity 联邦到云厂商 IAM。

## 四、AI Agent 身份

LLM Agent 大量调用 API，引出新身份问题：

| 问题 | 方案 |
|:---|:---|
| Agent 代表谁？ | OBO（On-Behalf-Of）token |
| 范围如何限制？ | 细粒度 scope + 时间 + 工具白名单 |
| 审计如何追溯？ | Token 内嵌请求上下文 |
| 撤销如何即时？ | 短期 token + 中心化撤销列表 |

新标准：**Model Context Protocol（MCP）** 中包含身份和权限设计。

## 五、Identity-First Security

```
传统安全:  边界（防火墙）+ 端点
        ↓
现代安全:  身份（IdP）+ 数据 + 行为
```

| 工具类型 | 代表产品 |
|:---|:---|
| IGA（身份治理） | SailPoint、Saviynt |
| PAM（特权管理） | CyberArk、Teleport |
| ITDR（身份威胁检测） | Microsoft Defender for Identity |
| IAM Posture | Permiso、Sonrai |

## 七、未来 3-5 年展望

| 方向 | 预测 |
|:---|:---|
| **Passkey 普及** | 消费应用 50% 支持，企业 30% |
| **零信任主流化** | 大企业必备，VPN 逐步退场 |
| **机器身份爆发** | 服务身份数量 > 人类身份 10 倍 |
| **AI Agent 身份** | MCP 等新协议落地 |
| **去中心化身份** | DID/W3C VC 在金融/医疗场景试点 |
| **国产化** | 信创身份产品成熟（统信、麒麟） |

## 九、企业落地路径

```
Step 1: 统一身份源 (HR + IdP 联动)
Step 2: 全员 SSO 接入 (邮箱/OA/办公套件)
Step 3: 强制 MFA (短信 → TOTP → WebAuthn)
Step 4: 应用接入 (开发/运维/数据库系统)
Step 5: 零信任接入 (替代 VPN)
Step 6: 机器身份 (服务间 mTLS)
Step 7: 持续治理 (审计、合规、自动化撤销)
```

## 十、学习路径

1. **协议**：OAuth/OIDC/SAML 三剑客必懂
2. **产品**：精通 Keycloak（自建）或 Okta（SaaS）
3. **集成**：K8s/Grafana/GitLab 等常用应用
4. **前沿**：WebAuthn、SPIFFE、零信任

> 💡 一个企业 IT 现代化的标志：**所有员工的所有系统访问都经过同一个 IdP**。
