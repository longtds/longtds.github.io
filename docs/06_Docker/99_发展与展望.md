# Docker 与容器发展展望

> Docker 公司在企业级失利、CNCF 接管标准制定，容器运行时正在多元化和专业化。

## 一、容器运行时演变

```
2013: Docker 一家独大
2015: OCI 标准 (Docker 把 runc 捐给 OCI)
2016: containerd 从 Docker 拆出
2017: K8s CRI 抽象
2020: K8s 宣布弃用 dockershim
2022: K8s 1.24 正式移除 dockershim
2024: containerd 成为 K8s 主流运行时
```

## 二、主流运行时对比

| 运行时 | 维护方 | K8s 集成 | 特点 |
|:---|:---|:---|:---|
| **containerd** | CNCF | CRI 原生 | K8s 主流 |
| **CRI-O** | Red Hat | CRI 原生 | OpenShift 默认 |
| **Docker** | Docker Inc. | 需 cri-dockerd | 开发场景 |
| **Podman** | Red Hat | 无 (Pod 原生) | 无 daemon、Docker 兼容 |
| **Kata** | 社区 | 通过 containerd | MicroVM 隔离 |
| **gVisor** | Google | 通过 containerd | 用户态 syscall |

## 三、镜像构建工具变革

```
Docker Build (传统)
   ↓ 慢、需要 daemon、root 权限
BuildKit (Docker 内置后端)
   ↓ 并行、缓存优化
Kaniko / Buildah (无 daemon)
   ↓ K8s 内构建镜像
nerdctl + BuildKit (containerd 原生)
   ↓ 完全替代 docker CLI
```

**生产推荐**：
- 开发本地：Docker Desktop / Podman Desktop
- CI 流水线：Kaniko / Buildah (无 daemon)
- K8s 内构建：BuildKit Pod

## 四、镜像格式演进

| 格式 | 用途 | 特点 |
|:---|:---|:---|
| OCI Image Spec v1 | 通用标准 | 当前主流 |
| zstd 压缩 | 大镜像 | 解压快 3 倍 |
| eStargz | 懒加载 | 启动快 80% |
| Nydus | 字节贡献 | 块级懒加载 |
| OverlayBD | 阿里贡献 | 块设备格式 |

**AI 模型镜像痛点**：单镜像 50-200GB，eStargz/Nydus 在 AI 场景价值巨大。

## 五、安全趋势

### 5.1 镜像签名（Sigstore）

```bash
# 签名
cosign sign --key cosign.key registry.io/app:v1

# 验证
cosign verify --key cosign.pub registry.io/app:v1
```

### 5.2 SBOM 与漏洞扫描

```bash
syft nginx:alpine -o cyclonedx-json > sbom.json   # 生成 SBOM
grype sbom:sbom.json                              # 扫描漏洞
trivy image nginx:alpine                          # 一站式
```

### 5.3 Rootless 容器

```bash
# Podman 默认 rootless
podman run -it ubuntu bash

# containerd rootless
containerd-rootless-setuptool.sh install
```

## 六、Wasm 容器

OCI 镜像 + Wasm 字节码（不是 Linux 二进制）：

| 项目 | 集成方式 |
|:---|:---|
| WasmEdge + containerd | runwasi shim |
| Spin (Fermyon) | spin-k8s |
| Krustlet | K8s kubelet 替代 |

**优势**：
- 启动 < 1ms
- 镜像 KB 级
- 跨架构（x86/ARM/RISC-V 一份代码）
- 沙箱安全（不共享内核）

## 七、未来 3-5 年展望

| 方向 | 预测 |
|:---|:---|
| **Docker CLI** | 仅作开发工具，生产被 nerdctl/podman 取代 |
| **containerd 一统** | K8s 90%+ 集群使用 |
| **镜像懒加载** | AI 大镜像场景普及 |
| **Rootless 默认** | 安全合规要求驱动 |
| **Wasm 容器** | 边缘 / 插件场景占 20% |
| **机密容器** | 金融/医疗强制要求 |

## 八、对运维的影响

| 变化 | 运维新要求 |
|:---|:---|
| 切换 containerd | 学 crictl/nerdctl 命令 |
| 镜像签名 | CI 集成 cosign，K8s 配置 admission |
| Rootless | userns 配置，存储驱动调整 |
| Wasm | runwasi 部署，新调试方法 |
| AI 大镜像 | 配置懒加载、并发拉取 |

## 九、命令对照表（Docker → 现代）

| 操作 | Docker | nerdctl (containerd) | crictl (K8s 节点) |
|:---|:---|:---|:---|
| 列出容器 | `docker ps` | `nerdctl ps` | `crictl ps` |
| 查看日志 | `docker logs` | `nerdctl logs` | `crictl logs` |
| 进入容器 | `docker exec` | `nerdctl exec` | `crictl exec` |
| 镜像列表 | `docker images` | `nerdctl images` | `crictl images` |
| 构建镜像 | `docker build` | `nerdctl build` | (不支持，用 buildkit) |

## 十、学习路径

1. **基础**：Docker 命令 + Dockerfile 精通
2. **进阶**：containerd + nerdctl + BuildKit
3. **生产**：镜像签名、SBOM、安全扫描
4. **前沿**：Wasm、Kata、Confidential Container

> 💡 Docker 是过去 10 年最伟大的开发者工具，但不会是下一个 10 年的生产运行时。
