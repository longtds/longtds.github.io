# 发展与展望

> Docker / 容器技术正在被重塑：**Docker Engine 战略收缩 + containerd 成事实标准 + 替代运行时(crun/youki/Kata/gVisor)百花齐放 + Wasm 容器扎根 + Confidential Containers 进生产 + Buildah/Kaniko/nerdctl 多构建栈 + GPU/MIG 标配 + 边缘/嵌入式爆发 + 国产 OCI(iSulad)崛起 + 供应链安全(SLSA/Sigstore)强制化**。本章给一份面向 2026-2030 的趋势判断 + 跟进路径。

## 一、Docker 公司战略

### 1.1 现状（2026）

```
Docker Inc:
  - Docker Desktop (商业，企业收费)
  - Docker Hub (公共镜像 + 企业 plan)
  - Docker Scout (镜像安全)
  - BuildKit / Compose Spec (开源)
  - Testcontainers (收购)
  
Docker 开源:
  - Engine (开源 Apache 2.0)
  - BuildKit / Buildx
  - containerd (CNCF 毕业，社区主导)
  - runc (CNCF 毕业)
  - Compose Spec (社区)

商业化重点:
  - 开发者本地（Desktop / Scout）
  - SaaS 镜像扫描
  - 不与 K8s 集群运行时直接竞争
```

### 1.2 5 年判断

```
✅ Docker Desktop 持续商业化（中小开发者）
✅ Docker Engine 在 CI / 单机 / 边缘 长期存活
✅ K8s 生态 100% containerd / CRI-O（dockershim 已退场）
✅ Docker Inc 不再是云原生中心
⚠️ Podman / nerdctl 在开发者侧蚕食 Docker CLI 份额
```

## 六、供应链安全强制化

### 6.1 SLSA / Sigstore / SBOM

```
SLSA (Supply chain Levels for Software Artifacts):
  L0  无保障
  L1  Provenance 文档
  L2  Hosted build + Provenance + signed
  L3  Hardened build + non-falsifiable
  L4  Two-person review + hermetic

Sigstore:
  - cosign 签名
  - rekor 透明日志
  - fulcio 短期证书
  - policy-controller K8s 准入

SBOM:
  - SPDX / CycloneDX 双标准
  - syft / Trivy / Anchore 生成
  - 漏洞溯源（log4shell 后强制）

监管:
  - 美国 EO 14028（联邦采购必备 SBOM）
  - 欧盟 CRA (Cyber Resilience Act, 2027 落地)
  - 中国《数据安全法》《关基保护条例》
```

### 6.2 5 年判断

```
2026-2027  SBOM + 签名 在金融/政府强制
2027-2028  SLSA L2+ 成大企业基线
2028-2030  全行业供应链审计透明化
```

## 十四、结语

```
容器技术已经 12 年。
从 LXC → Docker 横空 → CoreOS rkt 对抗 → OCI 标准 → containerd 胜出。
从单引擎 → 多运行时百花齐放 (runc/crun/kata/gVisor/youki/runwasi)。
从 docker-in-docker → K8s 原生 buildkit/kaniko。
从开发者工具 → 企业级平台 + 主权云。
从手工 Dockerfile → 多架构 + SBOM + 签名 + Provenance + sigstore。
从云端 → 边缘 → IoT → Wasm 极致下沉。
从国际方案 → 国产 OCI (iSulad) + 国产 GPU 容器 + 海光 CSV。

我们正处在加速期:
  - Docker 公司收缩，OCI 生态接管
  - containerd 成事实标准，低层运行时百花齐放
  - 机密容器从 PoC 进生产
  - Wasm 从边缘扎根，与容器共生
  - GPU 容器成 AI 默认
  - 供应链安全强制化 (SLSA + SBOM + Sigstore)
  - 国产化栈全面成熟
  - 边缘/嵌入式爆发

工程师该做的:
  1. 别只学 docker run，要懂 OCI 全栈 + 多运行时
  2. 把 Harbor + cosign + Trivy + SBOM 当成基本功
  3. 把 K8s containerd 当成首要场景
  4. 把 GPU + AI 容器列入必修
  5. 把 机密计算 + 国产化 + Wasm 提前储备
  6. 把 eBPF 可观测当成"容器之眼"
  7. 别只盯着 Docker 公司，盯 CNCF + OCI + sigstore + 国产开源
```

> 📖 **核心判断**：容器的未来 = **OCI 标准内多元运行时(containerd 主导 + crun/kata/gVisor/Wasm) + 机密容器 + GPU/AI 标配 + 供应链强签名 + 边缘爆发 + 国产 OCI**。把这六条线钉死，5 年内不愁。**Docker 不是核心，OCI 才是；Docker Engine 不是中心，containerd 才是；CLI 不重要，编排+安全+多架构才重要**。**别只学 docker run，要学 OCI 全栈 + 多运行时 + 供应链 + 机密 + 国产化** —— 这是平台架构师与运维工程师的分水岭。
