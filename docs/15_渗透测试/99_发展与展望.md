# 渗透测试发展与展望

> AI 既是攻击工具，也是防御工具。攻击面持续扩大（云 + AI + IoT），渗透测试进入自动化与智能化时代。

## 一、攻击面持续扩大

### 1.1 现代攻击面全景

```
传统:  网络边界 + Web 应用
   ↓
现代:  ┌── 云资源（S3/RDS 错配）
       ├── 容器/K8s（镜像/RBAC 错配）
       ├── 供应链（NPM/PyPI 投毒）
       ├── API（OWASP API Top 10）
       ├── 移动应用
       ├── IoT / OT / 工控
       ├── AI 模型（提示词/越狱）
       └── 内部员工（社工 + 钓鱼）
```

### 1.2 ASM（攻击面管理）

| 厂商 | 产品 |
|:---|:---|
| Microsoft Defender EASM | 微软 |
| Tenable.asm | Tenable |
| Randori | IBM |
| 360 安全大脑 | 国内 |
| 知道创宇 ZoomEye | 国内 |

## 二、自动化渗透

### 2.1 工具演进

```
2010s: Metasploit + 手动
2015s: nmap + Nuclei + 自动扫描
2020s: BAS（持续攻击模拟）
2024+: LLM + Agent 自主渗透
```

### 2.2 BAS（Breach and Attack Simulation）

| 工具 | 厂商 |
|:---|:---|
| Caldera | MITRE 开源 |
| Atomic Red Team | Red Canary |
| Cymulate | 商业 |
| AttackIQ | 商业 |

**思路**：用脚本模拟 ATT&CK 中的具体技术，验证防御能力。

```

   ↓
```

## 四、云原生渗透

### 4.1 攻击场景

| 场景 | 技术 |
|:---|:---|
| 云资源错配 | 公开 S3、暴露 RDS |
| K8s API 暴露 | 直接访问 kube-api |
| 容器逃逸 | privileged、hostPath、cgroup 滥用 |
| IMDS 利用 | 通过 SSRF 获取 IAM 凭证 |
| Service Account 滥用 | K8s 内权限提升 |

### 4.2 工具

| 工具 | 用途 |

| **ScoutSuite** | 多云审计 |
| **kube-hunter** | K8s 主动测试 |
| **kubeaudit** | K8s 配置审计 |
| **Peirates** | K8s 容器内渗透 |
| **Pacu** | AWS 利用框架 |

## 五、Web 攻击新热点

| 类型 | 趋势 |
|:---|:---|
| **API 安全** | OWASP API Top 10 2023 全面更新 |
| **GraphQL** | 注入、未授权、复杂度攻击 |
| **JWT/OAuth** | 算法混淆、密钥泄露 |
| **SSRF + 云元数据** | 持续高发 |
| **Server-Side Template Injection** | 多语言模板引擎 |
| **依赖混淆** | npm/PyPI 包仿冒 |

## 七、红蓝紫体系

```
红队 (Red Team):    模拟真实攻击者，长期渗透
蓝队 (Blue Team):   防御方，监测响应
紫队 (Purple Team): 双方协作，持续提升
```

**国内"实战攻防演练"**：
- 国家级护网行动（每年 6-9 月）
- 行业级（金融、电力、政府）
- 企业内部红蓝对抗

## 九、对运维 / 安全的影响

|:---|:---|

| 攻击面管理 | ASM 工具部署 |

## 十、学习路径

1. **基础**：网络/Web/Linux/Python
2. **靶场**：DVWA → VulnHub → HackTheBox → TryHackMe
3. **认证**：OSCP（实战金标准）
4. **专项**：选一个深度（云/AD/Web/AI）
5. **实战**：CTF 比赛 + 实际项目
6. **前沿**：AI 红队、云原生渗透

> 💡 渗透不是技术展示，是**让组织变安全的手段**。漏洞背后是修复，对抗背后是协作。
