## 八、持续测试（Continuous Testing）

```
理念:
  CI/CD 每步都有测试 (PR + Merge + Pre-prod + Prod)
  测试 = 流程, 不是阶段

实践:
  Pre-commit hook → 静态 + 单元 (< 30s)
  PR → 单元 + 集成 + 覆盖率 (< 5min)
  Merge → 契约 + 镜像 + SAST (< 15min)
  Pre-prod → E2E + 性能基线 + DAST (< 60min)
  Prod → Smoke + Synthetic 24/7
  Post-deploy → 灰度观测

工具:
  GitHub Actions / GitLab CI / Jenkins
  Argo Workflows + Argo CD
  Tekton Pipelines
  Buildkite
  CircleCI

提速:
  Test Selection (Launchable / Bazel)
  并行 (xdist / shard / matrix)
  Cache (Bazel / Nx / Turborepo)
  分布式 (K8s Job)

5 年判断:
✅ Continuous Testing 替代阶段化测试 (2026-2027)
✅ Test Selection 普及, CI 提速 50-70%
✅ Pre-commit 测试普及
```

  1. LLM 模型测试是未来 5 年最大机遇
  2. AI 加持自动化是分水岭 (Copilot / Codium)

  8. 鸿蒙 + 小程序 + 车机 多端兼容
