# 发展与展望

> K8s 正在被重塑：**控制平面瘦身 + Operator/CRD 成应用模型 + 多集群联邦(Karmada)主流 + Service Mesh Ambient 化 + 边缘 + AI 训练/推理调度 + KubeVirt 替代 VMware + Confidential Containers + eBPF 全栈可观测 + Wasm 共生 + 国产开源全面成熟 + Platform Engineering 重塑研发**。本章给一份面向 2026-2030 的趋势判断 + 跟进路径。

## 一、K8s 项目本身

```
版本节奏:   3 个月 1 个 minor (1.30→1.31→...)
LTS:        v1.31 起官方支持 LTS 试点 (3年长期支持)
Deprecation: dockershim (✓), beta API 持续退场

主线方向:
  - 控制平面瘦身 (一些扩展点 → CRD/Aggregated API)
  - kubelet → CRI-API + 标准化
  - Sidecar 内建 (1.28+)
  - Gateway API 替代 Ingress
  - Job/Indexed Job/JobSet 增强 (AI/批处理)
  - ResourceClass / DRA (Dynamic Resource Allocation) (1.30+)
  - 拓扑感知调度 + NUMA + 高性能计算
  - 镜像懒加载 (Stargz/Nydus)
  - Pod 级用户命名空间 (1.30 GA)
  - cgroup v2 全栈
  - 内置 KMS v2

退场:
  - Pod Security Policy (已退)
  - dockershim (已退)
  - 大量 beta apiVersion (持续)
```

## 二、Operator/CRD 成应用模型

```
"K8s is the new Linux kernel; CRD/Operator is the new package."

2026 现状:

  - KubeBlocks 国产数据库 Operator 统一接口 ⭐
  - vLLM Production Stack / Ollama / KServe AI 服务 Operator
  - Crossplane 把云资源也 CRD 化
  - Operator 互操作 (OLM v2 / Argo CD Native)

5 年判断:
✅ 90% 复杂应用通过 Operator 部署
✅ Operator 间标准化协议 (OpenAPI / OAM v2)
✅ AI / DB / 中间件 一律 Operator
✅ Crossplane + Cluster API 成基础设施新底座
```

```
应用:

  - Submariner / Skupper 跨集群 Service

5 年判断:
✅ 中大型企业 100% 多集群
✅ Karmada 国内占主导 (国产 + 华为/腾讯/阿里支持)
✅ Cluster API 成集群创建/升级标准
✅ 多云 (公有 + 私有 + 信创) 联邦成主流
```

## 四、Service Mesh Ambient 化

```
2024-2025:  Istio Ambient GA → sidecar 退场
2026:        Cilium SM / Linkerd 持续追赶
2027-2028:  Ambient 在大规模生产成主流

替代:
  - sidecar 模式资源开销大 (每 Pod +50MB+)
  - Ambient: ztunnel (L4) + waypoint (L7) 按需启用
  - 内存节省 60-90%
  - L7 治理仍可用

国产:
  - Higress (阿里) ⭐
  - Sermant (华为, Java Agent + Mesh 混合)
  - KubeSphere Service Mesh

5 年判断:
✅ Ambient 在大规模生产渗透 50%+
✅ Mesh + Gateway API 融合
✅ Cilium SM 在 eBPF 派系扎根
```

## 五、AI 调度成核心场景

```
2025-2027 关键演进:
  - DRA (Dynamic Resource Allocation, 1.30+) 替代 device plugin
  - JobSet + LeaderWorkerSet 多角色训练
  - PyTorch / vLLM / Ray Operator 标准化
  - HAMi 多 GPU 混调度 (国产)
  - Volcano / Kueue 批处理双轨
  - Topology-aware scheduler (NUMA / IB)
  - GPU 跨节点共享 (NVLink Switch / RDMA)

5 年判断:
✅ K8s 成 AI 训练/推理事实标准 (取代 SLURM 在企业场景)
✅ vLLM / SGLang / llm-d 推理三巨头
✅ 国产 GPU + HAMi 混调度成主流国产方案
✅ Confidential AI (机密 LLM) 进生产
```

## 十五、结语

```
K8s 已经 12 年。
从 Borg 致敬 → Docker Swarm/Mesos 对抗 → K8s 胜出 → CNCF 生态爆发。
从单集群 → 多集群联邦 → 跨云联邦 → 主权云。
从 sidecar → Ambient → eBPF 革命。
从容器 → VM (KubeVirt) → Wasm 三位一体。
从 device plugin → DRA → 智能调度。
从国际方案 → 国产开源全栈成熟 (KubeSphere + Karmada + HAMi + DeepFlow)。
从 DevOps → GitOps → Platform Engineering。

我们正处在加速期:
  - Operator/CRD 成应用模型
  - 多集群联邦主流 (Karmada)
  - Service Mesh Ambient 化
  - AI 训练 / 推理调度成核心
  - 边缘 / IoT 爆发
  - KubeVirt 替代 VMware
  - 机密计算 + AI 进生产
  - eBPF 全栈替代
  - 国产开源全面成熟
  - Platform Engineering 重塑研发

工程师该做的:
  1. K8s 不是终点，云原生才是
  2. Operator / GitOps / Platform Engineering 是研运一体新范式
  3. 多集群 + 多云 + 信创 + 边缘 是国内大企业必修
  4. AI 调度 + 推理 + 训练 是下一战场
  5. eBPF / Ambient / DRA / Wasm 是核心技术分水岭
  6. 国产开源 (KubeSphere/Karmada/HAMi/DeepFlow) 必懂
  7. 别只学 kubectl，要学 Operator + 调度 + 治理 + 平台 + 成本
```

> 📖 **核心判断**：K8s 的未来 = **Operator/CRD 应用模型 + 多集群联邦(Karmada) + Service Mesh Ambient + AI 调度(DRA/HAMi/vLLM) + KubeVirt + Confidential AI + eBPF 全栈 + Wasm 共生 + 国产开源主导 + Platform Engineering**。把这 10 条线钉死，5 年内不愁。**K8s 不会被替代，但围绕它的栈会全面重塑** —— 这是平台架构师与运维工程师的下一波分水岭。
