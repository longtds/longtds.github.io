## 一、虚拟化技术选型

### 1.1 虚拟化原理

```
虚拟化的本质: 在物理硬件之上加一层抽象, 将物理资源 (CPU/内存/磁盘/网络)
逻辑划分为多个独立单元, 每个单元可以运行独立的操作系统。

  ┌───────┐ ┌───────┐ ┌───────┐ ┌───────┐
  │  VM1   │ │  VM2   │ │  VM3   │ │  VM4   │  虚拟机 (各自独立 OS)
  │ CentOS │ │ Ubuntu │ │ Win    │ │ CentOS │
  └───┬───┘ └───┬───┘ └───┬───┘ └───┬───┘
      │         │         │         │
  ┌───▼─────────▼─────────▼─────────▼───┐
  │          Hypervisor (VMM)             │  虚拟化层
  │  CPU 虚拟化  内存虚拟化  I/O 虚拟化    │
  └───────────────┬───────────────────────┘
                  │
  ┌───────────────▼───────────────────────┐
  │            物理硬件                    │
  │  CPU  内存  磁盘  网卡  GPU           │
  └───────────────────────────────────────┘

  CPU 虚拟化:
    Intel VT-x / AMD-V (硬件辅助虚拟化)
    - 特权指令直接执行 (不需要二进制翻译)
    - EPT (Extended Page Tables) / NPT (Nested Page Tables)
      → 二级地址转换, 减少 VM-Exit 次数

  内存虚拟化:
    影子页表 (Shadow Page Table, 早期软件方案)
    EPT/NPT (硬件方案, 现代主流)
    → Guest 虚拟地址 → Guest 物理地址 → Host 物理地址

  I/O 虚拟化:
    全虚拟化 (QEMU 模拟硬件, 性能低)
    半虚拟化 (Virtio, Guest 需要驱动, 性能高)
    硬件直通 (PCI Passthrough / SR-IOV, 性能最高)
```

### 1.3 选型建议

| 场景 | 推荐 | 原因 |
|:---|:---|:---|
| 开源云平台 | **KVM** | OpenStack 基石, 完全开源 |
| 企业数据中心 | **VMware ESXi** | 成熟稳定, vMotion/vSAN/HA |
| 混合 Windows 环境 | **VMware ESXi** | Windows 兼容性最好 |
| 成本敏感 | **KVM** | 完全免费, Linux 内核内置 |
| SMB / 家庭实验室 | **Proxmox VE** | KVM + LXC, Web 管理, 免费 |
| 微软生态 | **Hyper-V** | 与 Windows/AD/Azure 深度集成 |
| 超融合 | **VMware vSAN** 或 KVM+Ceph | vSAN 简单, Ceph 灵活 |

## 二、KVM

### 2.1 KVM 架构

```
┌─────────────────────────────────────────────────────────┐
│                    KVM 架构                              │
│                                                         │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐              │
│  │   VM1    │  │   VM2    │  │   VM3    │              │
│  │  Guest OS│  │  Guest OS│  │  Guest OS│              │
│  │ (Linux)  │  │ (Windows)│  │ (Linux)  │              │
│  └────┬─────┘  └────┬─────┘  └────┬─────┘              │
│       │              │             │                      │
│  ┌────▼──────────────▼─────────────▼─────┐              │
│  │            QEMU/KVM                     │              │
│  │                                        │              │
│  │  ┌──────────────┐  ┌────────────────┐ │              │
│  │  │  QEMU         │  │  KVM 内核模块   │ │              │
│  │  │  (用户态)     │  │  (内核态)       │ │              │
│  │  │              │  │                │ │              │
│  │  │ - 设备模拟    │  │ - /dev/kvm     │ │              │
│  │  │  (磁盘/网卡)  │  │ - CPU 虚拟化    │ │              │
│  │  │ - 内存管理    │  │  (VT-x/AMD-V)  │ │              │
│  │  │ - 迁移控制    │  │ - 中断注入      │ │              │
│  │  │ - VNC/SPICE  │  │ - EPT/NPT      │ │              │
│  │  └──────┬───────┘  └───────┬────────┘ │              │
│  │         │ ioctl(/dev/kvm)  │           │              │
│  │         └───────┬──────────┘           │              │
│  └─────────────────┼──────────────────────┘              │
│                    │                                     │
│  ┌─────────────────▼──────────────────────┐              │
│  │           Linux 内核                     │              │
│  │  调度器  内存管理  网络栈  文件系统       │              │
│  └─────────────────┬──────────────────────┘              │
│                    │                                     │
│  ┌─────────────────▼──────────────────────┐              │
│  │           物理硬件                       │              │
│  │  CPU (VT-x/EPT)  内存  磁盘  网卡       │              │
│  └────────────────────────────────────────┘              │
│                                                         │
│  管理栈:                                                 │
│  ┌──────────────────────────────────────────────┐       │
│  │  virsh (CLI)  /  virt-manager (GUI)          │       │
│  │        ↓                                      │       │
│  │  libvirt (API 守护进程, 统一管理接口)          │       │
│  │        ↓                                      │       │
│  │  QEMU/KVM  /  LXC  /  Xen (后端)              │       │
│  └──────────────────────────────────────────────┘       │
└─────────────────────────────────────────────────────────┘

  KVM vs QEMU:
    KVM:  Linux 内核模块, 负责CPU/内存虚拟化 (硬件加速)
    QEMU: 用户态程序, 负责设备模拟 (磁盘/网卡/显卡)
    合体: QEMU 负责外围, KVM 负责核心 → 完整虚拟化

  libvirt:
    统一管理 API, 屏蔽底层差异 (KVM/Xen/LXC)
    提供 virsh CLI + virsh API (Python/C/Go 绑定)
```

### 2.2 环境准备

```bash
# === 检查硬件虚拟化支持 ===
# Intel: VT-x + EPT
# AMD:   AMD-V + NPT (RVI)

# 检查 CPU 虚拟化
grep -E '(vmx|svm)' /proc/cpuinfo | head -1
# vmx = Intel VT-x, svm = AMD-V

# 检查 EPT/NPT
grep -E '(ept|npt)' /proc/cpuinfo | head -1

# 如果没有输出, 需要在 BIOS 中开启:
# Intel: Virtualization Technology + VT-d
# AMD:   SVM Mode + IOMMU

# === 安装 KVM ===
# Rocky / RHEL 9
dnf install -y qemu-kvm libvirt libvirt-daemon \
    virt-install virt-manager virt-viewer \
    bridge-utils virt-top guestfish \
    libguestfs-tools

# Ubuntu 22.04 / 24.04
apt install -y qemu-kvm libvirt-daemon-system \
    virtinst virt-manager virt-viewer \
    bridge-utils virt-top libguestfs-tools

# 启动 libvirt
systemctl enable --now libvirtd

# 验证
virsh version
virsh list --all
lsmod | grep kvm
# kvm             120832  3
# kvm_intel       323584  0
# (AMD: kvm_amd)

# === 用户权限 ===
usermod -aG libvirt,kvm $USER
# 重新登录生效

# === 开启嵌套虚拟化 (可选, 在 VM 内运行 VM) ===
# Intel
echo "options kvm-intel nested=1" > /etc/modprobe.d/kvm.conf
modprobe -r kvm_intel
modprobe kvm_intel
cat /sys/module/kvm_intel/parameters/nested
# Y = 已开启

# AMD
echo "options kvm-amd nested=1" > /etc/modprobe.d/kvm.conf
modprobe -r kvm_amd
modprobe kvm_amd
```

### 2.3 网络配置

```bash
# === NAT 模式 (默认, VM 可访问外网, 外网不可访问 VM) ===
# libvirt 默认创建 virbr0 (NAT 桥)
virsh net-start default
virsh net-autostart default

# 查看默认网络
virsh net-info default
virsh net-dumpxml default
# 网段: 192.168.122.0/24
# DHCP: 192.168.122.2-254
# 网关: 192.168.122.1

# === 桥接模式 (VM 与宿主机同网段, 推荐) ===
# 创建网桥 br0, 绑定到物理网卡 eno1

# 方式一: nmcli (NetworkManager)
nmcli connection add type bridge con-name br0 ifname br0 \
    ipv4.method manual ipv4.addresses 192.168.1.10/24 \
    ipv4.gateway 192.168.1.1 ipv4.dns 192.168.1.1

nmcli connection add type bridge-slave con-name br0-port1 \
    ifname eno1 master br0

nmcli connection up br0

# 验证
ip addr show br0
brctl show
# bridge name     interfaces
# br0             eno1

# 方式二: 配置文件 (Rocky/RHEL)
cat > /etc/sysconfig/network-scripts/ifcfg-br0 << 'EOF'
DEVICE=br0
TYPE=Bridge
BOOTPROTO=static
IPADDR=192.168.1.10
PREFIX=24
GATEWAY=192.168.1.1
DNS1=192.168.1.1
ONBOOT=yes
EOF

cat > /etc/sysconfig/network-scripts/ifcfg-eno1 << 'EOF'
DEVICE=eno1
TYPE=Ethernet
BOOTPROTO=none
ONBOOT=yes
BRIDGE=br0
EOF

systemctl restart network

# === 创建 libvirt 网络 (基于 br0) ===
cat > /tmp/br0-network.xml << 'EOF'
<network>
  <name>br0</name>
  <forward mode='bridge'/>
  <bridge name='br0'/>
</network>
EOF

virsh net-define /tmp/br0-network.xml
virsh net-start br0
virsh net-autostart br0

# === Open vSwitch (高级网络, VLAN/VXLAN) ===
# 安装
dnf install -y openvswitch
systemctl enable --now openvswitch

# 创建 OVS 网桥
ovs-vsctl add-br ovs-br0
ovs-vsctl add-port ovs-br0 eno1

# 创建带 VLAN 的端口
ovs-vsctl add-port ovs-br0 vlan100 tag=100 -- set interface vlan100 type=internal
ip addr add 192.168.100.1/24 dev vlan100
ip link set vlan100 up

# libvirt 使用 OVS
cat > /tmp/ovs-network.xml << 'EOF'
<network>
  <name>ovs-br0</name>
  <forward mode='bridge'/>
  <bridge name='ovs-br0'/>
  <virtualport type='openvswitch'/>
</network>
EOF
virsh net-define /tmp/ovs-network.xml
virsh net-start ovs-br0
virsh net-autostart ovs-br0
```

### 2.4 存储配置

```bash
# === 存储池 (Storage Pool) ===
# libvirt 存储池类型: dir / fs / lvm / zfs / iscsi / nfs / rbd

# 1. 目录存储池 (最简单)
virsh pool-define-as default dir --target /var/lib/libvirt/images
virsh pool-build default
virsh pool-start default
virsh pool-autostart default

# 2. LVM 存储池 (高性能, 支持快照)
# 先创建 VG (参考 10-存储系统部署与配置.md)
pvcreate /dev/sdb
vgcreate vmvg /dev/sdb

virsh pool-define-as vmvg logical --source-name vmvg --target /dev/vmvg
virsh pool-build vmvg
virsh pool-start vmvg
virsh pool-autostart vmvg

# 3. NFS 存储池 (共享存储, 支持迁移)
virsh pool-define-as nfs-pool --type nfs --source-host 192.168.1.20 \
    --source-path /data/vmimages --target /var/lib/libvirt/nfs-pool
virsh pool-start nfs-pool
virsh pool-autostart nfs-pool

# 4. iSCSI 存储池 (企业级共享存储)
virsh pool-define-as iscsi-pool --type iscsi \
    --source-host 192.168.1.30 --source-dev iqn.2024-01.example:storage \
    --target /dev/disk/by-path
virsh pool-start iscsi-pool
virsh pool-autostart iscsi-pool

# 查看存储池
virsh pool-list --all
virsh pool-info default

# === 创建磁盘镜像 ===
# 创建 qcow2 镜像 (推荐, 支持快照/精简置备)
qemu-img create -f qcow2 /var/lib/libvirt/images/vm1.qcow2 50G

# 创建 raw 镜像 (性能略高, 不支持快照)
qemu-img create -f raw /var/lib/libvirt/images/vm1.raw 50G

# 查看镜像信息
qemu-img info /var/lib/libvirt/images/vm1.qcow2

# 转换格式
qemu-img convert -f raw -O qcow2 vm1.raw vm1.qcow2
qemu-img convert -f qcow2 -O raw vm1.qcow2 vm1.raw

# 调整镜像大小
qemu-img resize /var/lib/libvirt/images/vm1.qcow2 100G

# 创建带 backing file 的镜像 (链接克隆)
qemu-img create -f qcow2 -b base.qcow2 -F qcow2 clone1.qcow2
```

### 2.5 创建虚拟机

```bash
# === 使用 virt-install 创建 VM ===

# 1. 从 ISO 安装 Linux
virt-install \
    --name vm1-centos \
    --ram 4096 \
    --vcpus 2 \
    --disk path=/var/lib/libvirt/images/vm1.qcow2,size=50,format=qcow2,bus=virtio \
    --network bridge=br0,model=virtio \
    --os-variant rocky9 \
    --cdrom /iso/Rocky-9.4-x86_64-minimal.iso \
    --graphics vnc,listen=0.0.0.0,port=5901 \
    --noautoconsole

# 2. 从 ISO 安装 Windows (需 VirtIO 驱动)
virt-install \
    --name vm2-win2022 \
    --ram 8192 \
    --vcpus 4 \
    --disk path=/var/lib/libvirt/images/vm2.qcow2,size=100,format=qcow2,bus=virtio \
    --disk path=/iso/virtio-win.iso,device=cdrom \
    --network bridge=br0,model=virtio \
    --os-variant win2k22 \
    --cdrom /iso/Windows_Server_2022.iso \
    --graphics vnc,listen=0.0.0.0,port=5902 \
    --noautoconsole

# 3. 从模板克隆 (快速创建)
virt-clone \
    --original vm-template \
    --name vm3-clone \
    --file /var/lib/libvirt/images/vm3.qcow2

# 4. 导入已有磁盘
virt-install \
    --name vm4-import \
    --ram 2048 \
    --vcpus 2 \
    --disk path=/data/existing-vm.qcow2,format=qcow2 \
    --network bridge=br0,model=virtio \
    --os-variant ubuntu2204 \
    --import \
    --noautoconsole

# 查看 os-variant 列表
osinfo-query os

# === VNC 连接 ===
# 查看分配的 VNC 端口
virsh vncdisplay vm1-centos
# :1 → 端口 5901

# 远程连接 (SSH 隧道)
ssh -L 5901:127.0.0.1:5901 user@192.168.1.10
# 然后用 VNC 客户端连接 localhost:5901

# === 基本管理 ===
virsh list --all                   # 查看所有 VM
virsh start vm1-centos             # 启动
virsh shutdown vm1-centos          # 优雅关机
virsh destroy vm1-centos           # 强制关机 (相当于拔电源)
virsh reboot vm1-centos            # 重启
virsh suspend vm1-centos           # 暂停
virsh resume vm1-centos            # 恢复
virsh undefine vm1-centos          # 删除 (不删磁盘)
virsh undefine vm1-centos --remove-all-storage  # 删除 (含磁盘)

# VM 信息
virsh dominfo vm1-centos           # 基本信息
virsh domstate vm1-centos          # 状态
virsh domiflist vm1-centos         # 网卡列表
virsh domblklist vm1-centos        # 磁盘列表
virsh vcpuinfo vm1-centos          # vCPU 信息
virsh dumpxml vm1-centos           # 完整 XML 配置

# 编辑 VM 配置
virsh edit vm1-centos              # 编辑 XML (改内存/CPU/磁盘/网卡)
```

### 2.6 快照与备份

```bash
# === 快照 (qcow2 格式) ===

# 创建快照
virsh snapshot-create-as vm1-centos \
    --name snap-before-update \
    --description "更新前快照"

# 查看快照
virsh snapshot-list vm1-centos
virsh snapshot-info vm1-centos snap-before-update

# 恢复快照
virsh snapshot-revert vm1-centos snap-before-update

# 删除快照
virsh snapshot-delete vm1-centos snap-before-update

# === 完整备份 (离线) ===
# 1. 关闭 VM
virsh shutdown vm1-centos

# 2. 备份 XML 配置
virsh dumpxml vm1-centos > /backup/vm1-centos.xml

# 3. 备份磁盘
cp /var/lib/libvirt/images/vm1.qcow2 /backup/vm1-centos-$(date +%Y%m%d).qcow2

# === 完整备份 (在线, 使用快照) ===
# 1. 创建外部快照 (不中断运行)
virsh snapshot-create-as vm1-centos \
    --name backup-$(date +%Y%m%d) \
    --disk-only --atomic

# 2. 复制原始磁盘文件 (此时写入新磁盘, 原始只读)
cp /var/lib/libvirt/images/vm1.qcow2 /backup/

# 3. 合并快照 (恢复写入原始磁盘)
virsh blockcommit vm1-centos vda --active --pivot

# 4. 删除快照
virsh snapshot-delete vm1-centos backup-$(date +%Y%m%d) --metadata
```

### 2.7 热迁移

```bash
# === 热迁移 (Live Migration) ===
# 前提:
#   1. 源和目标主机 CPU 兼容 (同型号或嵌套)
#   2. 共享存储 (NFS/iSCSI/Ceph) 或 磁盘镜像同步
#   3. 网络互通, 相同网桥配置
#   4. libvirt SSH 免密互信

# 1. 配置源/目标主机 SSH 免密
ssh-keygen -t ed25519
ssh-copy-id root@192.168.1.12

# 2. 在目标主机创建同名 VM 定义
virsh dumpxml vm1-centos | ssh root@192.168.1.12 \
    "virsh define /dev/stdin"

# 3. 执行迁移 (共享存储)
virsh migrate --live --persistent --undefinesource \
    vm1-centos qemu+ssh://192.168.1.12/system \
    --verbose

# 4. 迁移 (非共享存储, 复制磁盘)
virsh migrate --live --persistent --undefinesource --copy-storage-all \
    vm1-centos qemu+ssh://192.168.1.12/system \
    --verbose

# 5. 迁移到本地存储 (指定目标磁盘路径)
virsh migrate --live --persistent --undefinesource \
    --copy-storage-all \
    --xml /tmp/vm1-target.xml \
    vm1-centos qemu+ssh://192.168.1.12/system

# 查看迁移进度
virsh domjobinfo vm1-centos

# 迁移参数调优
virsh migrate --live --persistent --undefinesource \
    --compressed --comp-methods zlib \
    --bandwidth 1000 \
    vm1-centos qemu+ssh://192.168.1.12/system
# --compressed:          压缩传输 (减少带宽, 增加 CPU)
# --comp-methods zlib:   压缩算法
# --bandwidth:           带宽限制 (MiB/s)
# --postcopy-bandwidth:  后复制带宽限制
```

### 2.8 性能优化

```bash
# === 1. CPU 优化 ===

# vCPU 绑定 (pinning, 固定到物理 CPU 核心)
virsh vcpupin vm1-centos 0 2        # vCPU 0 → 物理 CPU 2
virsh vcpupin vm1-centos 1 3        # vCPU 1 → 物理 CPU 3
virsh vcpuinfo vm1-centos           # 查看绑定

# CPU 拓扑 (NUMA 感知)
# 查看主机 NUMA 拓扑
numactl --hardware
virsh capabilities | grep -A 20 'topology'

# NUMA 绑定 (VM 内存绑定到 NUMA 节点)
# virsh edit vm1-centos
# <domain>
#   <numatune>
#     <memory mode='strict' nodeset='0'/>
#   </numatune>
#   <cpu mode='host-passthrough' check='none'>
#     <numa>
#       <cell id='0' cpus='0-1' memory='4' unit='GiB'/>
#     </numa>
#   </cpu>
# </domain>

# CPU 型号: host-passthrough (最佳性能, 不可迁移到不同 CPU)
# <cpu mode='host-passthrough' check='none'/>

# CPU 型号: host-model (兼容性好, 可迁移)
# <cpu mode='host-model' check='partial'/>

# === 2. 内存优化 ===

# 大页 (HugePages, 减少 TLB Miss)
# 计算大页需求: 2048 个 2MB 大页 = 4GB
echo 2048 > /proc/sys/vm/nr_hugepages

# 永久配置
echo "vm.nr_hugepages = 2048" >> /etc/sysctl.d/99-hugepages.conf
sysctl -p /etc/sysctl.d/99-hugepages.conf

# VM 使用大页 (virsh edit)
# <memoryBacking>
#   <hugepages/>
# </memoryBacking>

# 内存气球 (Balloon, 动态调整 VM 内存)
# virsh edit
# <memballoon model='virtio'/>
# 运行时调整:
virsh setmem vm1-centos 3G --live       # 调整为 3GB
virsh dommemstat vm1-centos             # 查看内存使用

# === 3. 磁盘优化 ===

# Virtio-blk (默认, 性能好)
# <disk type='file' device='disk'>
#   <driver name='qemu' type='qcow2' cache='none' io='native'/>
#   <source file='/var/lib/libvirt/images/vm1.qcow2'/>
#   <target dev='vda' bus='virtio'/>
# </disk>

# cache 模式:
#   none:    不缓存 (最安全, 性能好, 推荐)
#   writeback: 写缓存 (性能最高, 掉电丢数据)
#   writethrough: 读写都缓存 (安全, 性能低)
#   unsafe:  只缓存写, 不刷盘 (最快, 不安全)

# io 模式:
#   native:  Linux AIO (异步 IO, 性能好, 推荐)
#   threads: 线程池 IO (兼容性好)
#   io_uring: 新异步 IO (内核 5.1+, 性能最好)

# Virtio-SCSI (多盘场景, 支持热插拔/SCSI 命令)
# <disk type='file' device='disk'>
#   <driver name='qemu' type='qcow2' cache='none' io='native' discard='unmap'/>
#   <source file='/var/lib/libvirt/images/vm1.qcow2'/>
#   <target dev='sda' bus='scsi'/>
# </disk>
# <controller type='scsi' index='0' model='virtio-scsi'>
#   <driver queues='4'/>
# </controller>

# 磁盘直通 (最高性能, 不可迁移)
# <disk type='block' device='disk'>
#   <driver name='qemu' type='raw' cache='none' io='native'/>
#   <source dev='/dev/sdb'/>
#   <target dev='vda' bus='virtio'/>
# </disk>

# === 4. 网络优化 ===

# Virtio-net (默认, 性能好)
# <interface type='bridge'>
#   <source bridge='br0'/>
#   <model type='virtio'/>
# </interface>

# 多队列 (Multi-Queue, 多 vCPU 场景提升网络性能)
# <interface type='bridge'>
#   <source bridge='br0'/>
#   <model type='virtio'/>
#   <driver name='vhost' queues='4'/>   ← 4 个队列 (对应 vCPU 数)
# </interface>

# vhost-net (内核态网络后端, 减少 context switch)
# 默认开启, 确认:
lsmod | grep vhost_net

# SR-IOV (硬件虚拟化网卡, 性能最高)
# 1. BIOS 开启 SR-IOV
# 2. 创建 VF (Virtual Function)
echo 8 > /sys/class/net/enp175s0f0/device/sriov_numvfs
# 3. VF 直通给 VM (virsh edit)
# <interface type='hostdev' managed='yes'>
#   <source>
#     <address type='pci' domain='0x0000' bus='0xaf' slot='0x10' function='0x0'/>
#   </source>
# </interface>

# === 5. PCI 直通 (GPU/网卡/USB) ===

# 查看设备
lspci -nn | grep -i nvidia
# 01:00.0 VGA compatible controller [0300]: NVIDIA ... [10de:1e04]

# 隔离设备 (防止宿主机使用)
echo "options vfio-pci ids=10de:1e04,10de:1e07" > /etc/modprobe.d/vfio.conf
echo "blacklist nouveau" >> /etc/modprobe.d/blacklist.conf

# 重建 initramfs
dracut -f

# 重启
reboot

# 验证
lspci -nnk -s 01:00.0
# Kernel driver in use: vfio-pci

# VM 配置 (virsh edit)
# <hostdev mode='subsystem' type='pci' managed='yes'>
#   <source>
#     <address domain='0x0000' bus='0x01' slot='0x00' function='0x0'/>
#   </source>
# </hostdev>

# === 6. VM XML 优化模板 ===
# virsh edit vm1-centos
# <domain type='kvm'>
#   <!-- CPU -->
#   <vcpu placement='static'>4</vcpu>
#   <cpu mode='host-passthrough' check='none'>
#     <topology sockets='1' dies='1' cores='2' threads='2'/>
#   </cpu>
#
#   <!-- 内存大页 -->
#   <memoryBacking>
#     <hugepages/>
#     <locked/>             ← 锁定内存 (防止被 swap)
#   </memoryBacking>
#
#   <!-- 时钟 -->
#   <clock offset='utc'>
#     <timer name='rtc' tickpolicy='catchup'/>
#     <timer name='pit' tickpolicy='delay'/>
#     <timer name='hpet' present='no'/>   ← 关闭 HPET (性能差)
#   </clock>
#
#   <!-- 特性 -->
#   <features>
#     <acpi/>
#     <apic/>
#   </features>
# </domain>
```

```
    TPS (Transparent Page Sharing):
      相同内存页只存一份 (类似 KSM)

      ESXi 需要回收内存时, 气球膨胀, VM 主动释放内存

      压缩比 2-3x, 速度比磁盘 swap 快

    Host Swap:
      最后手段, 将 VM 内存页交换到磁盘
      性能严重下降
```

### 3.4 存储配置

```
ESXi 存储架构:

  ┌───────────────────────────────────────────────────┐
  │                   ESXi 主机                        │
  │                                                   │
  │  ┌──────────────────────────────────────────────┐ │
  │  │              VMFS (集群文件系统)               │ │
  │  │                                              │ │
  │  │  Datastore1 (本地磁盘)                        │ │
  │  │  ├── vm1/vm1.vmdk                             │ │
  │  │  ├── vm1/vm1-flat.vmdk                        │ │
  │  │  ├── vm2/vm2.vmdk                             │ │
  │  │  └── iso/                                     │ │
  │  │                                              │ │
  │  │  Datastore2 (iSCSI 共享存储)                   │ │
  │  │  ├── vm3/vm3.vmdk                             │ │
  │  │  └── vm4/vm4.vmdk                             │ │
  │  └──────────────────────────────────────────────┘ │
  │                                                   │
  │  存储适配器:                                       │
  │  ├── vmhba0:  本地 SATA/NVMe                      │
  │  ├── vmhba32: 软件 iSCSI                           │
  │  └── vmhba1:  FC HBA                              │
  └───────────────────────────────────────────────────┘

  VMFS (Virtual Machine File System):
    - 集群文件系统 (多 ESXi 同时读写)
    - 支持精简置备/厚置备
    - 支持快照
    - VMFS 6: 自动空间回收 (UNMAP)

  VMDK 格式:
    .vmdk      元数据文件 (描述磁盘配置)
    -flat.vmdk 实际数据文件

  磁盘置备模式:
    厚置备延迟置零 (Thick Provision Lazy Zeroed):
      预分配空间, 首次写入时置零
      创建快, 性能中等

    厚置备置零 (Thick Provision Eager Zeroed):
      预分配空间, 创建时全部置零
      创建慢, 性能最好, 支持 FT (容错)

    精简置备 (Thin Provision):
      按需分配空间
      创建最快, 节省空间, 性能略低
      需要监控空间使用 (避免超分耗尽)
```

```bash
# === VM 磁盘管理 ===

# -d thin:       精简置备
```

### 3.7 性能优化

```bash
# === 1. CPU 优化 ===

# NUMA 绑定
# vSphere Web Client → VM Settings → Hardware → CPU
#   - 勾选 "Expose virtual NUMA to guest OS"
#   - 勾选 "NUMA node affinity" → 选择 NUMA 节点

# CPU 亲和性 (Affinity)
# VM Settings → CPU → Scheduling Affinity
#   - 固定到物理 CPU 核心
#   - 注意: 与 DRS 冲突 (设置亲和性后 DRS 不会自动迁移)

# CPU Hot Add
# VM Settings → CPU → 勾选 "CPU Hot Add"
#   - 运行时增加 vCPU (需 OS 支持)
#   - 注意: 开启后 vNUMA 禁用 (影响性能)

# === 2. 内存优化 ===

# 预留 (Reservation)
# VM Settings → Memory → Reservation
#   - 预留物理内存 (ESXi 不对此 VM 做 swap)
#   - 性能最好, 但降低超分率

# 限制 (Limit)
# VM Settings → Memory → Limit
#   - 限制 VM 可用内存上限
#   - 用于控制资源使用

# 大页
# ESXi 默认对 VM 使用 2MB 大页
# 查看:
esxcli system settings advanced list -o /Mem/AllocGuestLargePage
# 开启:
esxcli system settings advanced set -o /Mem/AllocGuestLargePage -i 1

# === 3. 磁盘优化 ===

# 磁盘类型选择
# 性能: eagerzeroedthick > zeroedthick > thin
# 空间: thin < zeroedthick < eagerzeroedthick

# Storage I/O Control (SIOC)
# vCenter → Datastore → Properties → Storage I/O Control
#   - 开启后自动调整 I/O 优先级 (防止噪声邻居)

# 磁盘调度器
esxcli system settings advanced list -o /Disk/SchedQuantum
# 默认 8 (ms), 高吞吐场景可调高

# SSD 识别
esxcli storage core device list -d naa.xxx | grep "Is SSD"
# 如果 SSD 未被识别:
esxcli storage nmp satp rule add -s VMW_SATP_LOCAL --device naa.xxx --option=enable_ssd

# === 4. 网络优化 ===

# SR-IOV (硬件虚拟化网卡)
# ESXi → Host → Manage → Hardware → Network → SR-IOV
#   - 在 BIOS 开启 SR-IOV
#   - 在 ESXi 开启 SR-IOV (需重启)
#   - VM 使用 SR-IOV 网卡

# 网络卸载 (TSO/LRO)
esxcli system settings advanced list -o /Net/TcpipDefLROEnabled
# LRO (Large Receive Offload): 开启提升吞吐
# TSO (TCP Segmentation Offload): 默认开启

# 巨型帧 (Jumbo Frame)
# vSwitch → Properties → MTU = 9000
# 物理交换机也需要开启 Jumbo Frame

# === 5. 高级参数 ===
# ESXi 高级参数 (esxcli)
esxcli system settings advanced list

# 日志级别
esxcli system syslog config get
esxcli system syslog config set --log-level=info

# 交换文件位置
# VM Settings → Swapfile location
#   - 与虚拟机存储在同一目录 (默认)
#   - 放在指定 Datastore (共享存储, 加快 VM 启动)

# 电力管理 (高性能模式)
esxcli hardware power policy get
esxcli hardware power policy set --id 1
# 1 = 高性能 (关闭节能, CPU 不降频)
```

## 四、对比总结

### 4.1 KVM vs ESXi 全面对比

| 维度 | KVM | VMware ESXi |
|:---|:---|:---|
| 类型 | Linux 内核模块 | 独立 Hypervisor |
| 开源 | ⭐ 完全开源 | 闭源 |
| 成本 | ⭐ 完全免费 | 基础版免费, 高级功能付费 |
| 宿主机 | Linux (任意发行版) | 自带 (VMkernel) |
| 管理 | libvirt/virsh/virt-manager | vCenter/vSphere Client |
| 集中管理 | ⭐ oVirt / OpenStack | ⭐ vCenter (成熟) |
| 热迁移 | virsh migrate | ⭐ vMotion (最成熟) |
| 高可用 | oVirt / Keepalived | ⭐ HA (内置) |
| 负载均衡 | oVirt / OpenStack | ⭐ DRS (内置) |
| 容错 | 无 (可用 Keepalived 替代) | ⭐ FT (实时镜像) |
| 存储 | LVM/ZFS/NFS/iSCSI/Ceph | VMFS/vSAN/iSCSI/FC/NFS |
| 网络 | Linux Bridge/OVS | vSwitch/DVS |
| 超融合 | Ceph / GlusterFS | ⭐ vSAN (简单) |
| 内存优化 | HugePages/Balloon | TPS/Balloon/Compression |
| GPU 直通 | ⭐ VFIO | ⭐ DirectPath I/O |
| 嵌套虚拟化 | ⭐ 支持 | 支持 (需开启) |
| Windows 性能 | 好 | ⭐ 最好 |
| Linux 性能 | ⭐ 最好 | 好 |
| 容器集成 | ⭐ Kubernetes/KubeVirt | vSphere with Tanzu |
| 社区/文档 | ⭐ 丰富 | 商业支持 |
| 运维门槛 | 中 (需 Linux 技能) | ⭐ 低 (Web UI) |

### 4.2 存储方案对比

| 方案 | KVM | ESXi | 适用场景 |
|:---|:---|:---|:---|
| 本地存储 | qcow2/raw/LVM/ZFS | VMFS (本地) | 小环境/单机 |
| NFS | libvirt NFS Pool | NFS Datastore | 共享存储 (简单) |
| iSCSI | libvirt iSCSI Pool | iSCSI Datastore | 共享存储 (企业) |
| FC | libvirt FC Pool | FC Datastore | 高端存储 (SAN) |
| 超融合 | Ceph / GlusterFS | vSAN | 超融合架构 |
| 分布式 | Ceph RBD | vSAN | 大规模扩展 |

### 4.3 选型决策树

## 五、迁移

### 5.1 VMware → KVM 迁移

```bash
# === V2V (Virtual to Virtual) 迁移 ===

# 1. VMware 磁盘格式转换
#    .vmdk → .qcow2

# 关闭 VM
# 从 ESXi Datastore 下载 .vmdk 文件 (或 SCP)
scp root@esxi:/vmfs/volumes/Datastore1/vm1/vm1.vmdk /tmp/

# 转换 (需要 qemu-img)
qemu-img convert -f vmdk -O qcow2 /tmp/vm1.vmdk /var/lib/libvirt/images/vm1.qcow2

# 2. 创建 VM 定义
virt-install \
    --name vm1-migrated \
    --ram 4096 \
    --vcpus 2 \
    --disk path=/var/lib/libvirt/images/vm1.qcow2,format=qcow2,bus=virtio \
    --network bridge=br0,model=virtio \
    --os-variant rocky9 \
    --import \
    --noautoconsole

# 3. 安装 VirtIO 驱动 (在 VM 内)
# 如果原 VM 使用 VMware VMXNET3/SCSI, 需替换为 VirtIO
# CentOS/RHEL:
dnf install -y qemu-guest-agent
# 修改磁盘/网卡驱动为 virtio, 重启验证

# 4. 使用 virt-v2v (自动化工具, 推荐)
# 安装
dnf install -y virt-v2v

# 从 ESXi 迁移 (需 ESXi 开启 SSH)
virt-v2v -ic esx://esxi01.example.com?no_verify=1 \
    -ip /tmp/esxi-password \
    "vm1" \
    -o local -os /var/lib/libvirt/images

# 从 VDDK 迁移 (更可靠, 需 VMware VDDK)
export LIBGUESTFS_BACKEND=direct
virt-v2v -ic vpx://vcenter.example.com/Datacenter/Cluster/esxi01 \
    -ip /tmp/vcenter-password \
    "vm1" \
    -o local -os /var/lib/libvirt/images
```

## 七、故障排查

### 7.1 KVM 常见问题

| 问题 | 原因 | 解决 |
|:---|:---|:---|
| VM 无法启动 | 磁盘/网络配置错误 | `virsh dumpxml` 检查, 查看日志 |
| 性能差 | VirtIO 未启用 / cache=writeback | 使用 virtio 磁盘/网卡, cache=none |
| 迁移失败 | CPU 不兼容 / 磁盘不同步 | `--unsafe` 或 `--persistent-xml` |
| 网络不通 | 网桥配置错误 / iptables 拦截 | 检查 br0 / iptables FORWARD |
| 内存不足 | VM 超分 | 减少 VM 数量 / 使用 Ballooning |
| 快照过大 | 快照链过长 | 合并快照 `blockcommit` |

```bash
# 日志查看

cat /var/log/messages | grep -i kvm

virsh start vm1 --console          # 启动并看控制台

lsmod | grep kvm
```

### 7.2 ESXi 常见问题

| 问题 | 原因 | 解决 |
|:---|:---|:---|
| VM 紫屏 (PSOD) | 硬件故障/驱动 Bug | 检查硬件, 更新固件 |
| VM 卡顿 | CPU/内存/IO 瓶颈 | esxtop 定位, 限制噪声邻居 |
| Datastore 不显示 | 存储断开 / VMFS 损坏 | rescan, vmkfs 修复 |
| vMotion 失败 | 网络不通 / CPU 不兼容 | 检查 vMotion 网络, CPU 兼容性 |
| HA 不工作 | 心跳网络 / 隔离 | 检查管理网络, 配置隔离响应 |
| 空间不足 | 精简置备超分 | 扩容 Datastore, 删除快照 |
| 时间不同步 | NTP 未配置 | 配置 NTP, 安装 VMware Tools |
