# 最佳实践

> 服务器最佳实践 = **10 项金标 + 选型决策树(通用/数据库/AI/存储/边缘) + BOM 与 TCO 估算 + 厂商管理(招投标/服务 SLA) + 生命周期管理(采购→部署→运营→退役) + 多平台并存策略 + 国产替代落地 + 资本支出 vs 运营支出 + 绿色与碳合规 + 团队组织**。本章从"会选机"升级到"运营整个数据中心硬件资产"。

## 一、10 项金标准

```
1.  ✅ 业务驱动选型 (先定 SLA/性能/容量, 再选硬件)
2.  ✅ TCO 思维 (5 年总成本, 不光看采购价)
3.  ✅ 平台标准化 (机型 3-5 种, 避免 SKU 爆炸)
4.  ✅ 多厂商策略 (至少 2 家, 避免锁定)
5.  ✅ 固件基线 + 灰度 (Production Baseline + 1→10→100)
6.  ✅ BMC 自动化 (Redfish + 集中管理平台)
7.  ✅ 监控 + 告警全覆盖 (Prometheus + DCIM)
8.  ✅ 备件 5-10% (CPU/MEM/SSD/NIC/PSU/风扇关键件)
9.  ✅ RMA SLA 跟踪 (DOA < 0.5% / AFR < 2% / RMA < 7 天)
10. ✅ 国产替代渐进 (业务非核心 → 核心, 双轨过渡)
```

## 二、选型决策树

### 2.1 通用计算 (Web/微服务)

```
判断:
  - 业务: 无状态 / 容器化 / 弹性
  - QPS: 中高 (1K-100K/节点)
  - SLA: 99.95%+
  
推荐:
  机型     2U 双路 (Dell R760 / HPE DL380 / 浪潮 NF5280)
  CPU      Intel SR 6448Y (32 核) × 2 / EPYC 9354 (32 核) × 2
  内存      256-512 GB DDR5
  磁盘      2× M.2 480GB (系统) + 4× 3.84TB NVMe (容器卷)
  网卡      2× 25G (业务) + 2× 1G (Mgmt)
  电源      2× 1100W
  
特性:
  ☐ K8s 友好
  ☐ 重启容忍 (无状态)
  ☐ 满 PCIe 扩展灵活
```

### 2.2 数据库 (OLTP/OLAP)

```
OLTP (MySQL/Oracle/PG):
  CPU      高频 (3.5+ GHz) + 32-64 核 + 大 L3 缓存 (EPYC ⭐)
  内存      512GB-2TB ECC + 满通道 + 低延迟
  磁盘      8-12× NVMe 3.84TB Gen5 + Hardware RAID + BBU
  网卡      2× 25G + 2× 10G (主从复制)
  特点     CPU 频率 / 内存延迟 / NVMe IOPS 优先

OLAP (ClickHouse/Doris/StarRocks):
  CPU      多核 (96-128) + AVX-512
  内存      512GB-1TB
  磁盘      12-24× NVMe (容量大) + LZ4 压缩
  网卡      2× 100G (跨节点 shuffle)

关键:
  ☐ ECC + Chipkill (DB 容错)
  ☐ DWPD ≥ 3 (写密集)
  ☐ NUMA 绑核 + 内存隔离
  ☐ 双路 PSU + UPS
```

### 2.3 AI 训练 / 推理

```
训练 (8 卡):
  机型     8U HGX H100/H200/B200 (NVIDIA HGX 平台)
  CPU      2× Xeon SR Platinum (48-64 核) / EPYC 9554+
  内存      1-2 TB DDR5
  GPU      8× H100/H200/B200 SXM (HBM3/HBM3e)
  存储      8× NVMe Gen5 (本地 shuffle)
  网卡      8× IB CX-7 400G (东西) + 2× 25G (Mgmt)
  电源      6× 3000W
  散热      液冷 (强推 H200/B200)
  
推理:
  L40S × 4-8 / H100 / 国产昇腾 / DCU
  2U/4U 标卡 (PCIe)
  对应业务 SLA + 模型规模

国产:
  Atlas 800 (昇腾 910B)
  曙光 X785 (DCU)
  浪潮 NF5688G7 (H100/H200)
  
关键:
  ☐ NVLink + IB 全互联
  ☐ 液冷预算
  ☐ DCGM 监控 ⭐
  ☐ 整机柜可选 (NVL72 大规模)
```

### 2.4 存储

```
对象存储 (Ceph/MinIO):
  CPU      双路 16-24 核
  内存      128-256GB
  磁盘      36-72× HDD 22TB (主) + 2× NVMe (BlueStore DB+WAL)
  网卡      2× 25G + 2× 100G
  特点     盘多 + 容量大 + CPU 适中

分布式块 (Ceph RBD):
  All NVMe:  24-48× NVMe 7.68TB
  混合:      4× NVMe (缓存) + 12-24× HDD
  
文件 (Lustre/BeeGFS):
  MDS:   高 CPU + 大内存 (元数据)
  OSS:   多盘 + 高带宽 + IB/RoCE

NAS (NetApp/Qumulo/国产):
  专用机型 (厂商定制)
  双控热备 + SAS/NVMe 混合
  
关键:
  ☐ 盘位密度
  ☐ 双路 PSU
  ☐ Erasure Coding (节省空间)
  ☐ 缓存盘 NVMe 高 DWPD
```

### 3.3 TCO 5 年模型

```
CapEx (资本支出):
  ☐ 硬件采购    单台 × 数量
  ☐ 机柜 / DC 建设
  ☐ 网络设备 (Spine/Leaf 交换机)
  ☐ 软件许可

OpEx (运营支出, 5 年):
  ☐ 电费       功率 × 24 × 365 × 5 × $0.07-0.12/kWh
  ☐ 冷却电      电费 × 0.3 (PUE 1.3)
  ☐ 备件 + RMA  采购 × 3-5%/年
  ☐ 运维人力    工程师 × 工资 × 5
  ☐ 网络带宽
  ☐ 数据中心租金 / 机柜费

示例 (1000 台 AI 服务器, 5 年):
  CapEx: 1000 × $400K = $400M
  电费:  10kW × 1000 × 24 × 365 × 5 × $0.10 = $44M
  冷却:  +$13M (PUE 1.3)
  备件:  $20M
  人力:  $10M (10 人 × $200K × 5)
  ──────
  TCO 5y: ~$487M
  CapEx 占比 82% (硬件密集)
  
对比通用 (1000 台 2U):
  CapEx: 1000 × $25K = $25M
  电费:  0.5kW × 1000 × 24 × 365 × 5 × $0.10 = $2.2M
  TCO 5y: ~$35M
  CapEx 占比 71%
```

### 3.4 关键 TCO 优化

```
☐ 选高 PUE 数据中心 (1.1-1.2 vs 1.5+)
☐ 液冷 (省冷却电 30-50%)
☐ 多平台并存 (避免单一厂商溢价)
☐ 自研 / OCP-like (Hyperscale 必)
☐ 备件优化 (集中 + 共享 vs 分散)
☐ 寿命延长 (固件升级 + 大保养, 5→6-7 年)
☐ 二手处置 (回收价值 / 内部循环)
```

## 四、厂商管理

### 4.1 招投标

```
RFP 必含:
  ☐ 业务场景 + SLA + 性能基线
  ☐ 配置规格 (BOM 详细 + 接口标准)
  ☐ 兼容性要求 (OS / 中间件 / 国密)
  ☐ 服务 SLA (4h/8h/NBD + 备件)
  ☐ 备件 + 维保期 (3-5 年)
  ☐ 测试方法 (烤机 + 性能 + 兼容)
  ☐ POC 要求 (现场 / 远程)
  ☐ 培训 + 文档
  ☐ 价格 (设备 + 服务 + 维保, 5 年总价)
  ☐ 资质 (信创 + 等保 + 国密)
  
评分:
  技术 50%
  商务 30%
  服务 + 资质 20%

避坑:
  ❌ 唯低价 (忽略服务质量 + TCO)
  ❌ 单厂商绑定
  ❌ 维保期短 (3 年 → 后续天价)
  ❌ 不要求 SLA + 罚则
```

### 4.2 服务 SLA

```
分级:
  ☐ NBD (Next Business Day)        非核心
  ☐ 8h (8 小时备件)                重要业务
  ☐ 4h (4 小时备件 + 工程师)        核心生产 ⭐
  ☐ 2h / 1h (金融 / 政府关键)

罚则:
  ☐ 备件超时 (按小时 / 万元)
  ☐ 工程师超时
  ☐ 故障升级路径明确
  ☐ 季度 / 年度服务报告

代表厂商:
  Dell ProSupport Plus
  HPE Pointnext
  Lenovo Premier
  华为 Care 2.0
  浪潮 i 服
```

### 4.3 多厂商策略

```
推荐组合:
  通用 2U:    Dell + 浪潮 / HPE + 超聚变 (2 家)
  AI 8 卡:    浪潮 NF5688 + 超聚变 G5500 / 华为 Atlas (信创)
  存储:      Dell / 浪潮 / 华为 (按场景)
  
比例:
  主力 60-70% / 备份 20-30% / 试点 10%

避免:
  ❌ 单一厂商 100% (议价权 + 风险集中)
  ❌ > 5 个厂商 (运维成本爆炸)
```

## 五、生命周期管理

### 5.1 完整流程

```
[采购] ────────────────────────
  RFP → 评标 → 合同 → 备货

[到货] ────────────────────────
  来料验收 + DOA 排除

[部署] ────────────────────────
  来料体检 → 固件基线 → 72h 烤机 → 上架 → 网络 → 监控
  (详见 18 硬件测试)

[运营] ────────────────────────
  日常: 监控 + 告警 + 容量
  季度: 固件审视 + 灰度升级
  年度: 大保养 + 容量评估

[故障] ────────────────────────
  现象 → 工单 → 备件 → 替换 → 烤机 → 上线 → Postmortem

[退役] ────────────────────────
  评估 → 数据擦除 → 拆机 → 资产报废 → 厂商回收
  
[文档] ────────────────────────
  全程归档: 入网 / 升级 / 故障 / 退役
```

### 5.2 关键 KPI

```
质量:
  ☐ DOA < 0.5%
  ☐ AFR (年故障率) < 2%
  ☐ MTBF (平均故障间隔) > 100K 小时

服务:
  ☐ RMA 周期 < 7 天 (P3)
  ☐ 备件 4h 到货 (P1)
  ☐ SLA 达成 > 99%

成本:
  ☐ TCO 优化 (年化)
  ☐ 备件库存 < 10%
  ☐ 寿命延长 (5→6-7 年)

可用性:
  ☐ 计划停机 < 4h/年
  ☐ 非计划停机 < 1h/年
  ☐ 数据中心可用性 > 99.99%
```

## 八、CapEx vs OpEx

```
CapEx (传统自建):
  ✅ 长期成本低
  ✅ 数据可控
  ✅ 自主架构
  ❌ 前期投入大
  ❌ 折旧 5 年
  ❌ 弹性差

OpEx (公有云):
  ✅ 弹性 + 按需
  ✅ 无前期投入
  ✅ 厂商负担运维
  ❌ 长期总成本高
  ❌ 数据出境 (Sovereign 矛盾)
  ❌ 厂商锁定

混合 (推荐):
  - 基础负载: 自建 (CapEx)
  - 弹性峰值: 公有云 (OpEx)
  - AI 训练: 看场景 (大规模自建, 试点用云)
  - 灾备: 跨云 + 跨 DC

国内主流:
  阿里 / 腾讯 / 华为 / 移动云 / 电信天翼云
  自建优势: 央企 + 金融 + 政府 100% 自建为主
```
