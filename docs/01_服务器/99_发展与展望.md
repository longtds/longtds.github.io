# 服务器发展与展望

> 服务器正在经历自 x86 普及以来最大的一次架构变革：从单一通用计算转向 CPU+GPU+DPU 的异构协同。

## 一、当前技术趋势（2025-2026）

### 1.1 Chiplet 与 3D 封装

| 厂商 | 技术 | 代表产品 |
|:---|:---|:---|
| Intel | EMIB + Foveros | Xeon 6 (Granite Rapids) |
| AMD | Infinity Fabric | EPYC Turin (Zen 5) |
| NVIDIA | CoWoS-L | B200 Superchip |

**影响**：单芯片晶体管数量突破 1000 亿，良率提升 30%+。

### 1.2 CXL 内存池化

CXL（Compute Express Link）打破了"CPU 直连内存"的限制：

```
传统：每台服务器独占内存，碎片化严重
CXL：内存池化，按需分配 → 整机柜内存利用率从 50% 提升到 85%
```

代表产品：Samsung CMM-D、Micron CZ120、Astera Labs Leo。

### 1.3 DPU/IPU 卸载

将网络/存储/安全任务从 CPU 卸载到专用芯片，释放 30% CPU 算力。

| 厂商 | 产品 | 特点 |
|:---|:---|:---|
| NVIDIA | BlueField-3 | 400Gb 网络 + Arm 核 |
| Intel | IPU E2000 | 200Gb + FPGA |
| AWS | Nitro | 自研、定制集成 |

### 1.4 液冷普及

| 散热方式 | PUE | 功率密度上限 | 适用场景 |
|:---|:---:|:---:|:---|
| 风冷 | 1.5 | 30kW/机柜 | 通用 |
| 冷板液冷 | 1.15 | 80kW/机柜 | AI 训练 |
| 浸没液冷 | 1.05 | 250kW/机柜 | 超算/大规模 GPU |

国内政策（"东数西算"+ PUE ≤ 1.25）正在加速液冷渗透。

## 二、AI 服务器爆发

### 2.1 形态变化

```
传统服务器：1U/2U，CPU 为中心
       ↓
AI 服务器：4U/6U/8U，GPU 为中心
       ↓
AI 整机柜：NVL72（72 GPU × 18 节点 × 液冷）
```

### 2.2 关键互联技术

| 技术 | 带宽 | 用途 |
|:---|:---|:---|
| NVLink 5.0 | 1.8TB/s/GPU | GPU↔GPU 卡间 |
| NVLink Switch | 130TB/s/机柜 | 跨节点 GPU 直连 |
| InfiniBand NDR | 400Gb/s | 跨机柜 |
| RoCE v2 | 400Gb/s | 以太网替代方案 |

### 2.3 主流 AI 服务器

| 产品 | 配置 | 适用 |
|:---|:---|:---|
| NVIDIA DGX H100 | 8×H100 + 4×NVLink Switch | 训练 |
| NVIDIA GB200 NVL72 | 36×Grace + 72×B200 | 万亿参数训练 |
| 浪潮 NF5688 | 8×H100 SXM | 国内主流 |
| 华为 Atlas 900 | 昇腾 910B | 国产替代 |

## 三、ARM 服务器崛起

ARM 服务器市占率从 2020 年的 < 1% 上升到 2025 年的约 15%。

| 厂商 | 产品 | 部署案例 |
|:---|:---|:---|
| AWS | Graviton 4 | AWS EC2 主力 |
| Ampere | Altra Max（128 核） | 阿里、字节、Microsoft |
| NVIDIA | Grace（72 核 Neoverse V2） | GB200 |
| 华为 | 鲲鹏 920 | 国内政企 |

**优势**：每瓦性能优于 x86 25-40%，更适合云原生横向扩展场景。

## 四、国产化进程

| 厂商 | CPU | 架构 | 应用场景 |
|:---|:---|:---|:---|
| 海光 | 海光三号 | x86 兼容（AMD Zen 授权） | 通用 |
| 鲲鹏 | 鲲鹏 920 | ARMv8 自研 | 通用 + 信创 |
| 飞腾 | FT-2500/腾云 S5000C | ARMv8 自研 | 政企信创 |
| 龙芯 | 3C5000 | LoongArch 自研指令集 | 信创 |
| 申威 | SW26010 | Alpha 衍生 | 超算 |

**信创路线**：芯片→服务器→操作系统→数据库→中间件→应用，已全栈覆盖。

## 五、未来 3-5 年展望

| 方向 | 预测 |
|:---|:---|
| **存算一体** | 部分 AI 推理场景商用（如忆芯、知存） |
| **光互联** | CPO（共封装光学）逐步取代电信号传输 |
| **量子-经典混合** | 量子加速 + 经典服务器协同 |
| **能耗约束** | PUE 普遍要求 < 1.25，碳指标纳入采购 |
| **超异构架构** | CPU+GPU+DPU+FPGA 在同一封装内（如 GB200） |
| **国产替代** | 信创服务器在政府/金融/能源领域达 50%+ |

## 六、对运维的影响

| 变化 | 运维新要求 |
|:---|:---|
| 多 GPU 互联复杂 | NCCL/NVLink 拓扑诊断能力 |
| 液冷普及 | 漏液检测、冷却液运维 |
| CXL 内存池 | 跨节点内存管理工具 |
| DPU 卸载 | 学习 DOCA/IPU SDK |
| 异构调度 | K8s + Device Plugin + GPU Operator |
| 信创栈 | 国产服务器/OS/数据库的故障排查经验 |

## 七、推荐学习路径

1. **基础**：精通本章「CPU/内存/存储/网络」基础部分
2. **进阶**：深入「GPU 架构 + NVLink + NCCL」
3. **拓展**：CXL、DPU、液冷各选一个深入研究
4. **应用**：关注国产化和 AI 服务器选型

> 💡 服务器形态的变化最终都会反映到 K8s 的 Device Plugin、CNI、CSI 上，所以底层硬件理解越深，上层运维越从容。
