# 固件管理

## 概述

固件是固化在硬件中的底层软件，定期更新固件可以修复安全漏洞和提升稳定性。

## 需要更新的固件

| 固件类型 | 更新频率 | 风险 |
|:---|:---:|:---:|
| BIOS/UEFI | 按需 | 高 |
| BMC/IPMI | 按需 | 高 |
| 网卡固件 | 按需 | 中 |
| 磁盘固件 | 按需 | 中 |
| RAID 卡固件 | 按需 | 高 |
| GPU 固件 | 按需 | 中 |

## 更新原则

1. 先阅读 Release Notes，确认修复了什么问题
2. 在测试环境验证后再更新生产环境
3. 分批更新，不要全量同时更新
4. 确保有回滚方案
5. 更新过程不要断电
