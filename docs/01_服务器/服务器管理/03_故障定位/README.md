# 硬件故障定位

## 概述

硬件故障定位是服务器运维的基本功。通过系统化的排查方法，可以快速定位故障点。

## 故障定位流程

```
告警/故障现象
    │
    ▼
1. 确认故障范围（单台/多台/全机房）
    │
    ▼
2. 查看 BMC/iLO 告警日志
    │
    ▼
3. 检查指示灯（LED 告警灯）
    │
    ▼
4. 分析系统日志（dmesg/syslog/eventlog）
    │
    ▼
5. 运行诊断工具（vendor 专用工具）
    │
    ▼
6. 定位故障组件
```

## 常见故障现象

| 现象 | 可能原因 | 排查步骤 |
|:---|:---|:---|
| 无法开机 | 电源/主板/内存 | 检查电源指示灯，逐件最小化测试 |
| 频繁重启 | 内存/主板/过热 | 查看 BMC 事件日志 |
| 磁盘离线 | 磁盘/背板/线缆 | 检查 SMART 信息，尝试换槽位 |
| 网络不通 | 网卡/线缆/交换机 | 检查链路指示灯，替换线缆 |
| 性能下降 | 内存/CPU/磁盘 | 运行性能诊断工具 |
