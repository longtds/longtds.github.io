# 服务器选型

## 概述

根据业务场景选择合适的服务器配置，在性能、容量、成本之间取得平衡。

## 按场景选型

| 场景 | CPU | 内存 | 存储 | 网络 | GPU |
|:---|:---|:---|:---|:---:|:---:|
| 通用计算 | 16-32核 | 64-128GB | 2×SSD+4×HDD | 10GbE | 无 |
| 数据库 | 32-64核 | 256-512GB | NVMe × N | 25GbE | 无 |
| 虚拟化 | 32-64核 | 256-768GB | SSD+HDD 混搭 | 10-25GbE | 可选 |
| AI 推理 | 16-32核 | 256-1024GB | NVMe × 2 | 100GbE | A100/H100 |
| 存储 | 8-16核 | 64-128GB | HDD × 12+ | 10-25GbE | 无 |

## 选型原则

1. 先确定业务需求（CPU/内存/IO 的瓶颈）
2. 参考现有监控数据做容量规划
3. 预留 20-30% 的余量应对突发
4. 考虑标准化，减少服务器型号种类
