# 网络发展与展望

> 网络正从"以包为单位"走向"以应用为单位"，从"通用 CPU 转发"走向"专用硬件加速"。

## 一、数据中心网络速率演进

| 年份 | 主流速率 | 介质 | 应用 |
|:---|:---:|:---|:---|
| 2010 | 10G | SFP+ | 通用服务器 |
| 2015 | 25/40G | QSFP+ | 虚拟化 |
| 2018 | 100G | QSFP28 | 云数据中心 |
| 2022 | 200/400G | QSFP-DD | AI 训练 |
| 2025 | 800G | OSFP | 大模型 |
| 2027 | 1.6T | OSFP-XD | 下一代 AI |

## 二、Cilium 与 eBPF 网络

Cilium 已经成为云原生网络的事实标准：

| 能力 | 传统方案 | Cilium |
|:---|:---|:---|
| Pod 网络 | flannel/calico (iptables) | eBPF (绕过 netfilter) |
| Service 负载均衡 | kube-proxy iptables | eBPF (O(1) 查找) |
| 网络策略 | iptables L3/L4 | eBPF L3-L7 |
| 可观测性 | tcpdump | Hubble (流粒度) |

性能提升：吞吐 +30%，延迟 -40%，CPU -50%。

## 三、Service Mesh 演进

```
sidecar 模式（Istio 1.x）
   ↓ 性能/复杂度问题
Sidecarless（Istio Ambient/Cilium Service Mesh）
   ↓ eBPF L7 处理
内核态 Service Mesh（未来）
```

| 方案 | 模式 | 延迟开销 |
|:---|:---|:---|
| Istio Envoy sidecar | 每 Pod 双 hop | 1-2ms |
| Istio Ambient | 节点级 ztunnel | < 0.5ms |
| Cilium Service Mesh | eBPF + Envoy | < 0.3ms |

## 四、DPU/SmartNIC

| 厂商 | 产品 | 算力 | 网络 |
|:---|:---|:---|:---|
| NVIDIA | BlueField-3 | 16×Arm A78 | 400G |
| Intel | IPU E2200 | FPGA + Xeon-D | 400G |
| AWS | Nitro v5 | 自研芯片 | 200G+ |
| 阿里 | CIPU | 自研芯片 | 200G |

**用途**：把网络/存储/安全/虚拟化卸载到 DPU，CPU 100% 给业务。

## 五、RDMA 与 RoCE

```
RDMA (Remote Direct Memory Access) 三种实现:
├── InfiniBand (Mellanox/NVIDIA 私有，性能最强)
├── iWARP (基于 TCP，已边缘化)
└── RoCE v2 (基于 UDP/IP，主流方向)
```

**应用场景**：

| 场景 | 为什么需要 RDMA |
|:---|:---|
| AI 训练 | NCCL 跨机 AllReduce |
| 分布式存储 | Ceph BlueStore over RDMA |
| 高性能数据库 | TiDB/Polardb 副本同步 |
| HPC | MPI 通信 |

## 六、IPv6 普及与 SRv6

| 技术 | 价值 |
|:---|:---|
| IPv6 | 地址空间无限，端到端简化 |
| SRv6 | 用 IPv6 地址做"程序化路由" |
| SRH | Segment Routing Header 携带路径 |

中国电信 5G 核心网、运营商专线大规模部署 SRv6。

## 七、未来 3-5 年展望

| 方向 | 预测 |
|:---|:---|
| **速率** | 800G 主流，1.6T 进入数据中心 |
| **CPO** | 共封装光学，光信号直接进芯片 |

## 八、对运维的影响

| 变化 | 运维新要求 |
|:---|:---|
| eBPF 网络 | 学 Cilium 调试 (cilium monitor、Hubble) |
| Service Mesh | 流量管理、灰度发布、mTLS 排查 |
| DPU 卸载 | DOCA SDK / IPU SDK |
| RDMA | 拥塞控制、PFC、ECN 调参 |
| SRv6 | 取代 MPLS 的路径管理 |
| 400G+ 网络 | 光模块运维、误码率分析 |

## 九、学习路径

1. **基础**：OSI 七层 + TCP 拥塞控制
2. **云原生**：CNI/Service Mesh/eBPF
3. **高性能**：RDMA/DPU/SmartNIC
4. **新兴**：SRv6/CPO

> 💡 网络故障 80% 在物理层，但学习时间 80% 应该花在协议层。
