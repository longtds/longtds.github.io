# 12_AIOps · 概述

> AIOps = AI for IT Operations。用 ML/LLM 让运维更聪明、更主动、更省人力。

## 一、AIOps 发展简史

| 阶段 | 时期 | 特征 |
|:---|:---:|:---|
| 监控自动化 | 2010-2015 | Nagios/Zabbix，阈值告警 |
| 数据驱动 | 2015-2018 | ELK + 异常检测算法 |
| AIOps 兴起 | 2018-2022 | Gartner 提出，根因分析、智能告警 |
| LLM 加持 | 2023-至今 | GPT/RAG 改写运维体验 |

## 二、AIOps 能力矩阵

```
┌─── 数据接入 ───────────────────┐
│  指标 / 日志 / 链路 / CMDB     │
├─── 数据处理 ───────────────────┤
│  特征工程 / 时序对齐 / 标注     │
├─── 智能分析 ───────────────────┤
│  异常检测 / 根因 / 预测 / 关联   │
├─── 自动响应 ───────────────────┤
│  告警降噪 / 自动修复 / 推荐     │
└─── 人机交互 ───────────────────┘
   LLM 对话 / 知识库 / 推荐 Runbook
```

## 三、AIOps 五大场景

| 场景 | 价值 | 难度 |
|:---|:---|:---|
| **异常检测** | 自动发现指标异常 | 中 |
| **告警降噪** | 同一故障告警合并 | 中 |
| **根因分析** | 多维度定位故障源 | 高 |
| **故障预测** | 提前预警容量、性能问题 | 高 |
| **自动修复** | 自愈、智能扩缩、配置回滚 | 高 |
| **LLM 助手** | 自然语言查询、Runbook 推荐 | 中 |

## 五、关键算法

| 类型 | 算法 | 用途 |
|:---|:---|:---|
| **统计** | 3σ / Z-Score / EWMA | 阈值动态化 |
| **时序** | ARIMA / Prophet / DeepAR | 趋势预测 |
| **聚类** | DBSCAN / K-Means | 告警合并 |
| **图模型** | 图神经网络 | 服务依赖根因 |
| **LLM** | GPT-4 / Claude / DeepSeek | 自然语言运维 |

## 六、AIOps 落地路径

```
Step 1: 数据基础
        - 监控指标（Prometheus）
        - 日志聚合（Loki/ELK）
        - 链路追踪（OTel/Tempo）
        - CMDB / 拓扑

Step 2: 单点能力
        - 异常检测（先做几个核心指标）
        - 告警降噪（去除噪音）

Step 3: LLM 接入
        - 知识库 RAG
        - 故障助手（HolmesGPT）

Step 4: 闭环自动化
        - 自愈剧本
        - 容量预测 + 自动扩容
```

## 七、学习路径

1. **基础**：监控告警系统（Prometheus + Grafana）
2. **数据**：时序数据库 + ML 基础
3. **算法**：异常检测、根因分析、时序预测
4. **LLM**：RAG + Agent + HolmesGPT
5. **平台**：搭建一套企业 AIOps

> 📖 AIOps 不是替代 SRE，而是**让一个 SRE 顶十个**。
