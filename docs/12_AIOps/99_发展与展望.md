# 发展与展望

> AIOps 正在被 LLM 重塑：**可观测性统一(OTel 一统三栈) + eBPF 全栈零侵入 + 异常检测从规则到深度学习 + LLM RCA + ChatOps Agent + MCP 协议标准化 + 自愈从脚本到 Agent + 多云 + 信创 + AI Native AIOps**。本章给一份 2026-2030 趋势 + 跟进路径。

## 一、12 大趋势

```
1. OpenTelemetry 一统 (Metrics+Logs+Traces+Profiles)
2. eBPF 成可观测主流 (Pixie/Coroot/DeepFlow/Cilium)
3. VictoriaMetrics 替代 Prometheus + Cortex/Mimir
4. ClickHouse 大规模日志栈
5. 异常检测深度学习 (USAD/Transformer/KAN)
6. LLM RCA + RAG 知识库 (Postmortem)
7. ChatOps Agent (LangGraph + MCP)
8. 自愈从脚本到 Agent (RBAC + 审批)
9. SLO 驱动告警 (Sloth/Pyrra)
10. Chaos + 自动验证 (季度演练)
11. 信创国产化 (夜莺/DeepFlow/Qwen/Ascend)
12. AI Native AIOps (LLM 内置)
```

## 二、技术演进

### 2.1 OpenTelemetry 一统

```
2024:       OTel Metrics + Logs + Traces 标准
2025:       Profiles 加入 (Pyroscope 整合)
2026-2028:  OTel SDK 默认 (所有语言)
            Collector 成边车 / DaemonSet 标配
            兼容 Prometheus / Loki / Tempo / Jaeger
2028+:       Events / RUM / Synthetic 全统一
```

### 2.2 eBPF 普及

```
2024:       Pixie / Coroot / Cilium Hubble 进入主流
2025:       DeepFlow (国产) 商用爆发
2026:       eBPF 成 K8s 可观测默认
            零代码 + 全栈拓扑 + 实时
2028:       eBPF Agent + LLM 协同 (自动 RCA)
```

### 2.3 异常检测 → AI Native

```
2010s:      规则 + 阈值
2018-2022:  统计 + 经典 ML (Prophet / Isolation Forest)
2023-2025:  深度学习 (USAD / AnomalyTransformer / Darts)
2026-2028:  Foundation Model for Time Series ⭐
            TimeGPT / Lag-Llama / Moirai
            统一时序大模型 (类 GPT)
2028+:       多模态 (metrics + logs + traces 联合建模)
```

### 2.4 LLM RCA → Agent

```
2023:       LLM 告警摘要 (简单)
2024:       LLM + RAG 历史故障检索 (中级)
2025:       LangGraph Agent (工具调用 + 推理)
2026-2027:  MCP 协议标准化 + Agent 自愈
2028+:       Agentic AIOps (自主 OODA 循环)
            人类只看 KPI + 异常
```

### 2.5 SLO 驱动

```
2020-2023:  CPU/Memory 阈值告警 (噪音)
2024-2025:  SLO + Error Budget (Sloth/Pyrra) 普及
2026+:       SLO 自动学习 (业务自适应)
            自动调 budget + 自动 throttle
```

### 2.6 自愈进化

```
Phase 1 (2020-): 静态脚本 (重启 / 扩容)
Phase 2 (2023-): 规则引擎 (Webhook + 多步)
Phase 3 (2025-): LLM 推荐 SOP + 单步审批
Phase 4 (2027-): LangGraph Agent + 多步 + RBAC
Phase 5 (2029+): Agentic 自主 (高自治, 人类 override)
```

## 九、结语

```
AIOps 已经 5 年, LLM 的进入让它彻底重启。
从 监控告警 → 可观测性 → AIOps → ChatOps Agent → Agentic AIOps。
从 静态阈值 → 动态阈值 → 深度学习 → Foundation Model。
从 人工 RCA → 拓扑辅助 → LLM + RAG → Agent 自主推理。
从 静态脚本 → Webhook → 规则引擎 → LangGraph + MCP → Agentic。
从 SLO 驱动 → Error Budget → 自适应 → AI Native。
从 单体监控 → 多云 → 信创 → 主权可观测。
从 工程师手动 → AIOps 辅助 → Agent 协同 → Agent 自治。

我们正处在 AIOps 大爆炸:
  - 工具: OTel + eBPF + VM + CH + Coroot + DeepFlow
  - 模型: Prophet → USAD → Foundation Model for TS
  - LLM: Qwen + DeepSeek + Claude + GPT-5
  - 协议: MCP 标准化工具
  - 平台: 夜莺 + 观测云 + 阿里 + 华为 + 国产 SaaS
  - 信创: Ascend + Qwen + DeepFlow
  - 监管: 等保 + 国密 + 备案

工程师该做的:
  1. 别停在 Prom + Grafana - 必修 OTel + eBPF
  2. 异常检测 + 深度学习 + Foundation Model 是必修
  3. LangGraph + MCP + Agent 是下一波分水岭
  4. 国产 (夜莺 + DeepFlow + Qwen) 是央企必修
  5. SLO 驱动告警 + Chaos 演练 是基本功
  6. Postmortem Blameless + 知识库 RAG 是闭环
  7. Coding Agent (Cursor) + AIOps Agent 协同
  8. 不学 LLM 的 SRE 5 年内被淘汰
```

> 📖 **核心判断**：AIOps 的未来 = **OTel 一统 + eBPF 默认 + VM/CH 大规模 + Foundation Model for TS + LLM RCA + LangGraph + MCP + Agent 自愈 + SLO 驱动 + Chaos 季度 + 国产化(夜莺/DeepFlow/Qwen) + Sovereign 可观测**。把这 10 条线钉死，5 年内不愁。**AIOps 不会取代 SRE，但 AI Native SRE 会取代不懂 LLM 的 SRE** —— 这是这一代运维工程师最大的机遇与分水岭。
