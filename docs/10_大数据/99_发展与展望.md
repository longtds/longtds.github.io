# 大数据发展与展望

> 大数据正在被 AI 重塑。下一代大数据 = 湖仓一体 + 实时 + AI Ready。

## 一、湖仓一体（Lakehouse）成为主流

### 1.1 三大开放表格式

| 格式 | 主要厂商 | 特点 |
|:---|:---|:---|
| **Apache Iceberg** | Netflix/Apple/AWS/Snowflake | 元数据高效、扩展性强、社区最活跃 |
| **Apache Hudi** | Uber/Onehouse | 增量处理、CDC 友好 |
| **Delta Lake** | Databricks | Databricks 生态强、ACID 完整 |
| **Paimon** | 阿里 | 流式数据湖，Flink 主推 |

**2024-2025 趋势**：
- Snowflake 拥抱 Iceberg（2024）
- Databricks 收购 Tabular（Iceberg 母公司）
- 三大格式开始 **统一互操作（XTable）**

### 1.2 现代大数据架构

```
                    BI / AI / 报表
                         ↑
        ┌──── 查询引擎层 ─────────┐
        │  Trino  ClickHouse  Spark │
        └────────────────────────┘
                         ↑
        ┌──── 元数据 Catalog ──────┐
        │   AWS Glue / Unity / Polaris  │
        └────────────────────────┘
                         ↑
        ┌──── 表格式层 ──────────┐
        │   Iceberg / Hudi / Delta  │
        └────────────────────────┘
                         ↑
        ┌──── 对象存储 ──────────┐
        │   S3 / OSS / MinIO / Ceph │
        └────────────────────────┘
```

## 二、计算引擎演进

### 2.3 Trino / Presto

```
Presto (FB 2012)
   ↓ 2018 分叉

```

**Trino** 已成为多源联邦查询的事实标准。

## 三、流处理新范式

### 3.1 Streaming Lakehouse

```
传统:   Kafka → Flink → 数据库（结果输出）
              ↓
现代:   Kafka → Flink → 数据湖（Paimon/Iceberg）
              ↑
              直接当作流读取、批读取
```

### 3.2 实时分析数据库

| 产品 | 特点 |
|:---|:---|
| Apache Pinot | LinkedIn 开源，毫秒级 OLAP |
| Apache Druid | 老牌实时 OLAP |
| ClickHouse | 列存王者，实时 + 历史合一 |
| Apache Doris | 通用实时仓库 |

## 四、数据集成现代化

### 4.1 ELT 取代 ETL

```
ETL (传统):  抽取 → 转换 → 加载
              ↑ 转换在中间，复杂
ELT (现代):  抽取 → 加载 → 转换
              ↑ 转换在仓内，SQL 友好
```

### 4.2 工具栈

| 类别 | 代表 |
|:---|:---|
| 数据集成 | Airbyte、Fivetran、Estuary |
| CDC | Debezium、Flink CDC、Maxwell |
| 编排 | Airflow、Dagster、Prefect |
| 转换 | dbt（事实标准）、SQLMesh |
| 治理 | Atlan、DataHub、Marquez |

### 5.2 向量 + 大数据

```

                ↓

```

## 七、未来 3-5 年展望

| 方向 | 预测 |
|:---|:---|
| **湖仓一体普及** | 80% 新建数据平台采用 |
| **Iceberg 统一** | 成为开放表格式事实标准 |
| **Hadoop 退场** | Hive 用 Iceberg 替换，HDFS 用 S3 替换 |
| **流批一体** | Flink/Spark 都支持，应用普及 |
| **AI 一体化** | 数据平台内嵌训练/推理能力 |
| **国产化** | Doris/StarRocks/OceanBase 进入主流 |
| **Serverless 化** | EMR Serverless 等按需付费成主流 |

## 八、对运维的影响

| 变化 | 运维新要求 |
|:---|:---|
| 湖仓化 | S3 + Iceberg + Trino 部署运维 |
| Spark on K8s | 抛弃 YARN，转 K8s 调度 |
| Flink CDC | 实时数据管道运维 |
| AI 集成 | 模型 + 数据混合工作负载 |
| 国产平台 | Doris/StarRocks/OceanBase 专项 |
| 成本治理 | S3 + 计算分时计费，账单优化 |

## 九、学习路径

1. **基础**：HDFS + Hive + Spark 三大件
2. **进阶**：Flink + Kafka + Trino
3. **现代**：Iceberg + dbt + Airflow
4. **AI 时代**：Databricks 思路 + 向量化

> 💡 Hadoop 时代结束了，但**大数据处理量级**才刚刚开始爆发。掌握新栈才能 hold 住未来 10 年。
