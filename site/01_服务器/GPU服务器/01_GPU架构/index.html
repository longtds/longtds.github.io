    GPU 架构¶

    GPU 从图形渲染走向 AI 计算核心。NVIDIA 占据 90%+ 市场，AMD 紧追，国产 GPU 在制裁倒逼下快速崛起。

    一、GPU 架构全景¶

    ┌──────────────── 国际厂商 ────────────────┐
│   NVIDIA: Volta → Hopper → Blackwell    │
│   AMD:    CDNA → CDNA3 → CDNA4 (MI300)  │
│   Intel:  Ponte Vecchio → Gaudi 3       │
│   Google: TPU v4/v5e/v5p (内部使用)      │
└─────────────────────────────────────────┘

┌──────────────── 国产 AI 芯片 ────────────┐
│   华为:      昇腾 910B/910C/910D         │
│   寒武纪:    思元 590/690                 │
│   海光:      DCU K100/K200 (CUDA 兼容)   │
│   摩尔线程:  MTT S4000/S5000             │
│   沐曦:      曦云 C500/C600              │
│   燧原:      邃思 2.0/3.0                │
│   昆仑芯:    昆仑 P800 (百度)             │
│   壁仞:      BR100/BR104                 │
│   天数智芯:  天垓 100/智铠 100            │
└─────────────────────────────────────────┘

    二、NVIDIA GPU 演进¶

    架构 时间 代表 制程 关键特性
    Volta 2017 V100 12nm Tensor Core 初代、NVLink 2
    Turing 2018 T4 12nm RT Core、INT8
    Ampere 2020 A100/A10 7nm MIG、TF32、第三代 NVLink
    Ada Lovelace 2022 L40/L4 4nm 推理优化
    Hopper 2022 H100/H200 4nm Transformer Engine、FP8、NVLink 4
    Grace Hopper 2023 GH200 4nm CPU+GPU 一体
    Blackwell 2024 B100/B200 4nm 第二代 Transformer Engine、FP4
    Blackwell Ultra 2025 B300 — 显存升级、性能 +50%
    Rubin 2026（规划） R100 3nm HBM4

    2.1 核心组件¶

    组件 作用
    CUDA Core 通用计算（FP32/FP64）
    Tensor Core 矩阵运算加速（AI 核心）
    RT Core 光线追踪（图形/渲染）
    HBM 高带宽显存（HBM3/HBM3e）
    NVLink GPU 间高速互联
    NVSwitch 多 GPU 拓扑交换

    2.2 中国特供版¶

    由于美国出口管制，NVIDIA 推出中国特供：

    型号 对应原版 限制
    A800 A100 NVLink 带宽降至 400GB/s
    H800 H100 NVLink 带宽降至 400GB/s
    H20 H100 算力大幅缩水（约 H100 的 15%）
    B20 B200 （2025 规划）

    ⚠️ 2024 年后 H800 也被禁，H20 是当前可合规进口的最强 NVIDIA 卡。

    三、AMD GPU（数据中心）¶

    架构 产品 时间 特点
    CDNA MI100 2020 首款数据中心 GPU
    CDNA 2 MI250X 2021 Frontier 超算（全球第一）
    CDNA 3 MI300A/X 2023 192GB HBM3，与 H100 直接竞争
    CDNA 3 MI325X 2024 256GB HBM3e
    CDNA 4 MI350/MI400 2025-2026 持续追赶

    软件栈 ROCm：AMD 的 CUDA 替代品，PyTorch/vLLM 已原生支持。生态比 CUDA 弱但快速进步。

    四、国产 AI 芯片（重点）¶

    4.1 华为昇腾（最重要）¶

    产品 制程 算力 (FP16) 显存 对标
    昇腾 910A 7nm 320 TFLOPS 32GB HBM V100
    昇腾 910B 7nm+ ~400 TFLOPS 64GB HBM2e A100
    昇腾 910C 7nm ~800 TFLOPS 128GB HBM3 H100 (~80%)
    昇腾 910D（规划） — — — H200

    软件栈： - CANN（Compute Architecture for Neural Networks）—— 对标 CUDA - MindSpore —— 华为自研框架 - MindIE —— 推理引擎（vLLM 类似定位） - PyTorch 适配：通过 torch_npu 插件

    重要应用： - 昇腾 Atlas 800 服务器（8 卡 910B/C） - Atlas 900 SuperCluster（万卡集群） - 国产大模型主力训练平台（DeepSeek 部分版本、智源、商汤、讯飞）

    4.2 寒武纪¶

    产品 时间 定位
    思元 270 2019 推理
    思元 370 2021 训推一体
    思元 590 2022 训练，对标 A100
    思元 690 2024 训练，对标 H100

    软件栈：Neuware（含 BANG 编程模型 + 编译器 + 算子库） 生态：PyTorch / TensorFlow 都有适配插件

    4.3 海光 DCU（CUDA 兼容路线）¶

    产品 制程 特点
    深算一号（K100） 7nm 类 A100
    深算二号（K200） 7nm 性能提升

    软件栈：DTK（DCU Toolkit）—— 高度兼容 CUDA - 大部分 CUDA 代码可"零修改"或微调编译运行 - 这是海光最大的差异化优势 - 来源：AMD GCN 架构授权 + 自研

    适配现状：vLLM、SGLang、PyTorch 已有官方版本。

    4.4 摩尔线程（MThreads）¶

    产品 定位 算力
    MTT S2000 2022 推理 —
    MTT S3000 2023 训练 —
    MTT S4000 2024 训练 接近 A100
    MTT S5000 2025 训练 对标 H100

    亮点： - 唯一国产全功能 GPU（同时做图形 + 计算） - 软件栈 MUSA，MUSIFY 工具可自动翻译 CUDA 代码 - KUAE 智算集群（千卡级别）

    4.5 沐曦（MetaX）¶

    产品 定位
    曦思 N100 推理
    曦云 C500 训练，对标 A100
    曦云 C600 训练，对标 H100

    软件栈：MACA（兼容 CUDA 编程模型） 特点：团队多来自 AMD，架构借鉴 CDNA

    4.6 燧原科技¶

    产品 定位
    邃思 1.0 (T10/T11) 推理
    邃思 2.0 (T20/T21) 训练，对标 A100
    邃思 3.0 训练 H100 级
    云燧 i20 推理卡

    软件栈：TopsRider 应用：腾讯系（腾讯持股）、上海仪电

    4.7 昆仑芯（百度）¶

    产品 制程 应用
    昆仑 1 (818) 14nm 内部使用
    昆仑 2 (R200) 7nm 文心一言推理
    昆仑 P800 7nm 文心 4.0 训练

    特点：百度自研，文心大模型生态深度绑定

    4.8 壁仞科技¶

    产品 算力（FP16） 状态
    BR100 1024 TFLOPS（理论） 实体清单受限
    BR104 256 TFLOPS 已量产

    特点：发布时号称"性能超 H100"，但被列入实体清单后供应链受阻。

    4.9 天数智芯¶

    产品 定位
    天垓 100 训练（对标 A100）
    智铠 100 推理

    软件栈：兼容 CUDA 生态

    4.10 国产 GPU 对比总表¶

    厂商 训练旗舰 生态策略 软件栈 自主度 推荐度
    华为昇腾 910C 自建 CANN 极高 ⭐⭐⭐⭐⭐
    海光 K200 CUDA 兼容 DTK 中（AMD 授权） ⭐⭐⭐⭐
    寒武纪 思元 690 自建+适配 Neuware 高 ⭐⭐⭐⭐
    摩尔线程 S5000 CUDA 翻译 MUSA 高 ⭐⭐⭐
    沐曦 曦云 C600 CUDA 兼容 MACA 中 ⭐⭐⭐
    燧原 邃思 3.0 自建 TopsRider 高 ⭐⭐⭐
    昆仑芯 P800 内部主导 XPU SDK 高 ⭐⭐⭐（百度生态强）
    壁仞 BR100 自建 BIRENSUPA 高 ⭐⭐（供应受限）
    天数智芯 天垓 100 CUDA 兼容 — 中 ⭐⭐

    六、显存与互联¶

    6.1 HBM 演进¶

    类型 带宽（每堆栈） 容量 应用
    HBM2 256 GB/s 8GB V100
    HBM2e 460 GB/s 16GB A100
    HBM3 819 GB/s 24GB H100
    HBM3e 1.2 TB/s 36GB H200 / B200
    HBM4（2026+） 1.6+ TB/s 48GB 下一代

    6.2 GPU 互联技术¶

    技术 带宽 用途
    PCIe 4.0 64 GB/s 通用
    PCIe 5.0 128 GB/s 新一代
    NVLink 4 900 GB/s NVIDIA H100 内部
    NVLink 5 1.8 TB/s NVIDIA Blackwell
    HCCS (华为) 392 GB/s 昇腾 910 内部
    Infinity Fabric (AMD) 800 GB/s MI300 内部

    七、国产 GPU 选型建议¶

    场景 1: 信创强制 + 训练
   → 昇腾 910B/C（生态最完善、华为生态护航）

场景 2: 信创强制 + 推理
   → 昇腾 910B 或 海光 DCU 或 寒武纪 590

场景 3: 想用 CUDA 代码迁移最少
   → 海光 DCU（DTK 兼容性最好）
   → 摩尔线程 MTT（MUSIFY 工具）

场景 4: 大规模训练（千卡以上）
   → 昇腾 Atlas 集群（最成熟）

场景 5: 推理服务（vLLM）
   → 昇腾 910B + vllm-ascend（首选）
   → 海光 DCU（次选）

场景 6: 仍可买 NVIDIA
   → H20（最强可合规进口）
   → 二手 H800/A100（注意来源）

    八、国产 GPU 常见坑¶

    坑 建议
    算子缺失 提前用 profiler 跑模型，统计未覆盖算子
    多机互联 国产卡 RDMA 适配比 NVIDIA 弱，需厂商支持
    驱动不稳定 锁定测试通过的驱动版本，谨慎升级
    精度问题 FP16/BF16 数值表现可能与 NVIDIA 有微小差异
    生态滞后 新模型发布后，国产卡通常滞后 2-8 周适配
    调试工具弱 类 Nsight 工具普遍缺失，定位性能瓶颈难
    混部困难 NVIDIA + 国产 GPU 共存集群运维复杂

    九、产业现状（2025）¶

    维度 数据
    中国 AI 芯片市场规模 约 ¥1500 亿/年
    NVIDIA 中国市占率 从 90% → 50%（被特供 + 国产挤压）
    昇腾市占 约 25%（国产第一）
    其他国产合计 约 25%
    国家政策 "信创目录" + "国产替代率"考核
    主要采购方 三大运营商、阿里/字节/腾讯、政府智算中心

    十、未来 3 年趋势¶

    方向 预测
    昇腾持续领先 910C/D 替代部分 A100/H100 需求
    CUDA 兼容路线 海光/摩尔/沐曦持续投入翻译工具
    生态收敛 国产框架/推理引擎统一适配主流国产卡
    HBM 国产化 长鑫/长江存储突破 HBM3
    训练能力突破 万卡国产集群成为现实（昇腾已实现）
    推理优势 国产 GPU 在推理场景率先实现替代

    十一、运维查看命令¶

    # NVIDIA

# 寒武纪

mthreads-gmi

# 通用：PCIe 设备查看
lspci | grep -i -E "nvidia|huawei|hygon|cambricon|moore|metax"

    十二、学习路径建议¶

    传统 NVIDIA 路线（仍主流）:
   CUDA 编程基础 → cuBLAS/cuDNN → TensorRT/vLLM → 分布式（NCCL/Megatron）

国产 GPU 路线（信创必备）:
   昇腾 CANN 入门 → MindSpore 或 PyTorch+NPU
   → MindIE 或 vllm-ascend → Atlas 集群运维

通用 AI 运维路线:
   理解多家硬件差异 → 推理引擎调优（vLLM/SGLang）
   → K8s + Device Plugin → 多卡监控与故障排查

    💡 核心判断：未来 3-5 年中国 AI 集群是"NVIDIA + 昇腾 + 海光 + 其他"多元并存的格局。SRE 必须至少掌握 NVIDIA + 昇腾两种生态，否则在信创项目中会非常被动。
